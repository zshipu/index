Notion 3.0：Agent 功能详解

Notion 在9月19日发布了 3.0 版本，其中最引人注目的就是全新的 Agent 功能。

这篇文章将从实用角度出发，带你深入了解 Notion 3.0 能为我们带来什么。

打造你的专属“个人Agent”

每个人心里都有一千个哈姆雷特，而 Notion Agent 就像你的专属助手，你需要做的只是下达任务，它就能为你高效完成。

如何让你的队友更加懂你？

如何让它成为你最得力的助手，能够第一时间帮你省时省力？

你可以为你的个人Agent进行个性化，给他设置形象和人设。

Notion Agent为每个用户的Agent分配了一个页面，其中包含自定义其行为的说明。此页面就像一个记忆库，可以帮助Agent更快地完成工作。

你可以自定义Agent，在这样的一个页面里面配置代理身份、对话互动、记忆库等内容，这将直接影响Agent在后面执行任务的效果。

只需要告诉它如何分类任务、设置响应格式以及在开始工作之前需要参考哪些信息。

随着使用Agent越来越多，它就会越来越理解你的使用习惯，成为你最得心应手的工作伙伴。

不过局限的是，目前仅仅支持个人用户定义自己的agent，团队协作agent预计将在下一版本呈现。

你可以在Agent市场看到不同的模板，有很多大神分享的提示词人设，以筛选出的免费模板为例。

强大的多步骤工作流

Notion Agent 的能力远不止于简单的问答和编辑，它可以跨越数百个页面，进行长时间（20分钟以上）的自主工作。这为我们带来了令人惊叹的效率提升。

案例1：一键创建新员工入职计划

设想一下，你需要为新加入AI团队的工程师李华创建一个入职指南。过去，这需要你手动创建页面，整理信息，耗时耗力。

现在，你只需要给 Notion Agent 一个指令：

为即将加入AI团队担任AI应用工程师的新工程师李华创建一份入职指南。指南内容应包含需要见面的人员数据库、需要准备的工具清单以及重要团队页面的链接。

Notion Agent 会立刻为你生成一个新页面，其中包含了内置数据库、待办事项列表以及所有可用模板，大大简化了入职流程。

案例2：秒速搭建复杂 CRM 系统

另一个让我惊艳的案例，是使用 Notion Agent 构建一个复杂的 CRM 系统。过去需要数小时甚至数天才能完成的任务，现在只需几分钟。这极大地节省了时间，让你能更专注于客户本身。

只需将下面的完整指令交给 Notion Agent：

通过添加后端切换来创建 Notion CRM，创建四个数据库（公司、联系人、机会、合同），然后添加指定的关系、汇总、视图和示例数据，以验证一切是否按预期工作。  
  
完整提示：  
# CRM 数据库系统设置说明  
  
你的任务是创建一个Notion 中的客户关系管理 (CRM) 系统就在本页。该系统将帮助您追踪公司、联系人、销售机会和合同。请遵循以下说明，按照指定的顺序，以确保正确的数据库关系和功能。  
  
## 第一阶段：页面结构设置  
  
### 步骤 1A：创建后端切换  
在当前页面上，创建：  
- 标题为“🗃️ 后端”的切换块  
- 在切换按钮内添加一段：“这是我们保存 CRM 数据库的地方”  
  
### 步骤 1B：创建数据库结构  
在“🗃️后端”切换按钮中，您将创建  
  
4 个数据库作为子页面：  
1. 公司数据库  
2. 联系人数据库  
3. 机会数据库  
4. 合同数据库  
  
## 第 2 阶段：数据库创建（仅限核心属性）  
  
### 步骤 2A：创建公司数据库  
创建一个名为“Companies”的新数据库  
  
作为后端切换内的子页面  
  
具有以下属性：  
-姓名（称谓）- 公司名称  
-地位（选择）- 选项：“活跃”、“潜在”、“不活跃”、“丢失”  
-行业（选择）- 选项：“科技”、“医疗保健”、“金融”、“制造”、“零售”、“教育”、“其他”  
-公司规模（选择）- 选项：“启动（1-10）”、“小型（11-50）”、“中型（51-200）”、“大型（201-1000）”、“企业级（1000+）”  
-网站(URL) - 公司网站  
-电话（电话号码）- 公司主要电话  
-地址（富文本）- 公司地址  
-描述（富文本）- 公司描述和注释  
-标签（多选）- 选项：“热门线索”、“战略”、“企业”、“SMB”、“合作伙伴”、“竞争对手”  
-创建日期（创建时间）- 自动填充  
-最后联系（日期）- 最后一次互动日期  
  
重要的：暂时不要添加任何关系属性。请先完成所有数据库的基本属性。  
  
### 步骤 2B：创建联系人数据库  
创建一个名为“联系人”的新数据库作为后端切换内的子页面  
  
具有以下属性：  
-姓名（称谓）- 联系人全名  
-电子邮件（电子邮件）- 主要电子邮件地址  
-电话（电话号码）- 直拨电话号码

-联系方式（选择）- 选项：“主要”、“次要”、“技术”、“财务”、“执行”

## 阶段 3：数据库关系设置  
  
### 步骤 3A：添加公司联系人关系  
1. 前往联系方式数据库  
2. 添加一个名为“公司”（关系类型）  
3. 选择公司数据库作为关系目标  
4. 选择“在公司上展示”并命名“联系人”  
5. 这会在公司和联系人之间建立双向关系  
  
### 步骤 3B：添加公司-机会关系  
1. 前往机会数据库  
2. 添加一个名为“公司”（关系类型）  
3. 选择公司数据库作为关系目标  
4. 选择“在公司上展示”并命名“机遇”  
  
### 步骤 3C：添加联系人-机会关系  
1. 前往机会数据库  
2. 添加一个名为“主要联系人”（关系类型）  
3. 选择联系方式数据库作为关系目标  
4. 选择“在联系人中显示”并命名“机遇”  
  
### 步骤 3D：添加机会-合同关系  
1. 前往合同数据库  
2. 添加一个名为“相关机会”（关系类型）  
3. 选择机会数据库作为关系目标  
4. 选择“展示机会”并命名“合同”  
  
### 步骤 3E：添加公司合同关系  
1. 前往合同数据库  
2. 添加一个名为“公司”（关系类型）  
3. 选择公司数据库作为关系目标  
4. 选择“在公司上展示”并命名“合同”  
  
### 步骤 3F：添加汇总属性  
1. 前往公司数据库  
2. 添加“管道总价值”属性（汇总类型）  
- 关系：“机会”  
- 属性：“交易价值”  
- 计算：“总和”  
3. 添加“赢得交易的价值”属性（汇总类型）  
- 关系：“机会”  
- 属性：“交易价值”  
- 计算：“总和”  
4. 添加“有效合约价值”属性（汇总类型）  
- 关系：“合同”  
- 属性：“合同价值”  
- 计算：“总和”

AI 正在以火箭般的速度无缝融入我们的工作和生活。

两年前的AI，还像一个蹒跚学步的婴儿，只会简单的对话和生成图片。而现在的AI，已经发展到让人惊叹的地步。
