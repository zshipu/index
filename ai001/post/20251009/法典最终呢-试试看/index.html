随着 Claude Code 的出现，Claude 的 Max 计划（100 美元或 200 美元/月）被许多人使用。
然而，在2025年8月底左右，突然发生了一些事情。 没错，“ClaudeCode-kun 变成白痴”事件。
因此，Codex CLI 备受关注，因 ChatGPT-5 的发布而引发了人们的期待。 它会被建立为ClaudeCode的替代品吗？ 为了寻找解决方案，我们的研究团队前往亚马逊深处。

起先

随着 ChatGPT-5 的发布，“Codex CLI”作为一种新选项而备受关注。 在本文中，我们将研究 Codex CLI 的 Plus 计划以评估其效用。 该验证是在基于截至 2025 年 9 月 7 日的功能的特定开发环境中进行的。 因此，请记住，结果可能取决于环境和项目条款。

TL; 博士

对于商业用途，Claude Code 在当前的团队开发和生产运营中仍然具有优势。 对于个人发展、小型项目和原型设计，Codex Plus 计划可以提供足够的生产力提升。

Codex Plus 计划挑战

在试用了大约一周的 Codex CLI（Plus 计划）后，发现了以下两个问题。

1. 工具执行中权限管理的粒度

第一个挑战是工具（命令）执行权限控制粒度的粗略性。 Claude Code 允许您设置一个列表以自动允许或拒绝仅某些命令，但 Codex CLI 尚未实现此功能。 虽然有一种非交互模式，允许一次性执行所有命令，但这与“选择并只允许受信任的命令”的作不同，在实践中使用是有风险的。

2. 上下文约束和提案范围狭窄

其次，Plus 计划上下文窗口的约束以及它如何影响提案的质量。

与 Pro 计划相比，Plus 计划中可以处理的信息量有限，因此 AI 建议的更正仅限于错误，并且往往是本地化的。 因此，有时我们觉得很难在考虑到跨多个文件的复杂修改、整个产品的设计理念以及与调用者的 I/O 契约的情况下提出大局的更改。 由于 Pro 计划提供了 128k 代币的广阔上下文窗口，我们相信这一挑战可能会得到解决。

总结及未来展望

在本文中，我们研究了 Codex CLI Plus 计划的实际使用情况，并考虑了其技术挑战。 总之，虽然 Plus 计划对个人发展很有用，但它在团队发展中的实施仍然是一个重大挑战。 具体来说，对工具执行权限的粗制滥造是一种安全隐患，上下文窗口的约束会使建议本地化，损害整个代码的一致性。 后者可以通过 Pro 计划解决，但只要前一个问题仍然存在，我们认为很难在实践中推荐使用当前的 Plus 计划。

我们预计未来的更新将增强权限管理能力，并为更多开发者解锁 Pro 计划的功能。

当然，这些只是截至 2025 年 9 月 7 日的评估。 Codex CLI 仍处于起步阶段，我们希望未来的更新能够解决此处提到的问题。
