53AI创始人/腾讯云(TVP)最具价值专家

2025 年 10 月 7 日，OpenAI 开发者大会（OpenAI DevDay 2025）如期而至，这场盛会不仅揭开了 OpenAI 过去两年的亮眼成绩单 ——400 万开发者、8 亿 ChatGPT 周活用户、API 每分钟 60 亿 token 消耗量，更以 “颠覆式工具矩阵” 为核心，集中发布了 AgentKit、Codex 正式版、ChatGPT 内置应用与 Apps SDK 等一系列重磅产品，彻底重构了 AI 开发的效率边界与应用场景。

AgentKit：让 “智能体开发” 告别 “从零开始”

在本次大会的所有发布中，AgentKit无疑是最受瞩目的焦点。作为一套面向开发者与企业的完整智能体工具集，它直接解决了过去智能体开发 “技术门槛高、落地周期长、安全难保障” 的三大痛点，甚至被业内人士评价为 “可能改写 AI 创业格局” 的关键产品。

AgentKit 的核心竞争力源于三大模块化组件：

  • Agent Builder

    可视化的 “智能体搭建画布” 是其核心亮点。开发者无需复杂编码，通过拖拽逻辑节点、连接工具、配置安全规则，就能快速构建多智能体工作流。例如，客服场景中可一键串联 “用户意图分类智能体”“信息查询智能体”“投诉处理智能体”，并内置预览运行、版本控制功能，大幅缩短迭代周期。更关键的是，它自带开源的 “Guardrails（安全护栏）”，能自动屏蔽个人信息（PII）、检测 “越狱” 攻击，为智能体加上 “安全锁”。

  • Connector Registry

    解决了智能体 “数据孤岛” 问题。在统一管理面板中，开发者可一键接入 Dropbox、Google Drive、Microsoft Teams 等预置数据源，也能集成第三方工具，让智能体直接调用企业内部数据，无需反复开发数据接口。

  • ChatKit

    让智能体 “无缝嵌入” 现有产品。通过这套工具，开发者可将聊天式智能体植入 APP、官网，并自定义品牌风格，目前已被 HubSpot 用于客户服务、Evernote 用于知识检索、LegalOn 用于法律文档咨询。OpenAI 透露，某电商平台通过 ChatKit 搭建的售后智能体，已处理了 67% 的客服工单，人力成本降低近 50%。

从可用性来看，ChatKit 与评估功能已全面开放，Agent Builder 进入公开测试阶段，且所有工具均包含在标准 API 定价中，极大降低了企业的使用门槛。

Codex 正式版：编码协作进入 “全场景互联” 时代

继 2025 年 5 月以研究预览版亮相后，Codex 正式版此次全面上线，不仅巩固了其 “编码协作伙伴” 的定位，更通过三大新功能实现了 “全场景覆盖”：

  • Slack 集成

    开发者无需切换工具，在 Slack 频道或线程中就能直接向 Codex 分配任务，例如 “优化这段用户登录接口代码”“生成数据库表结构文档”，响应速度与在编辑器中一致。

  • Codex SDK

    将 Codex CLI 的核心能力开放给第三方。开发者可将其嵌入自有工作流，基于 GPT-5-Codex 实现 “零微调” 的先进编码性能，某 SaaS 公司通过该 SDK 集成到内部开发平台后，代码生成效率提升了 40%。

  • 管理员工具

    为企业级用户提供 “精细化管控”。ChatGPT 工作区管理员可通过环境控制、监控面板，查看团队成员的 Codex 使用频率、功能调用类型，还能设置 “代码安全检查规则”，避免漏洞代码进入生产环境。

数据印证了 Codex 的爆发式增长：自 8 月初以来，其日活跃用户增长超 10 倍，GPT-5-Codex 上线三周内处理了 40 万亿 token；在 OpenAI 内部，几乎所有工程师都在使用 Codex，每周合并的 PR 数量增加 70%，且关键代码缺陷检出率提升至 92%。从 Duolingo 这样的初创公司，到思科、乐天等巨头，Codex 已成为编码流程中不可或缺的一环。

ChatGPT 内置应用 + Apps SDK：8 亿用户生态的 “双向奔赴”

OpenAI 此次的另一大突破，是让 ChatGPT 从 “对话工具” 升级为 “应用平台”——ChatGPT 内置应用正式推出，用户可在聊天界面中直接调用第三方服务：

  • 旅游场景：输入 “Booking.com帮我找巴黎 11 月 21-24 日带停车场的双人酒店”，ChatGPT 会直接返回带评分、价格的酒店列表，支持跳转预订；

  • 设计场景：向 Figma 发送 “把这个用户旅程图转化为流程图”，几秒内就能生成可编辑的 Figma 文件；

  • 学习场景：通过 Coursera 调用 “推荐适合初学者的 Python 课程”，还能直接查看课程大纲与报名链接。

首批上线的应用包括Booking.com、Canva、Expedia、Spotify 等，除欧盟地区外，所有 ChatGPT 用户（Free 到 Pro 版本）均可使用。而这一切的背后，是Apps SDK（预览版） 的支撑 —— 基于开放的 Model Context Protocol（MCP）标准，开发者可同时设计应用的逻辑与交互式界面，且 SDK 已开源。

更具吸引力的是，通过 Apps SDK 开发的应用，能触达 ChatGPT 的 8 亿用户；今年晚些时候，OpenAI 还将推出 “应用目录”，并开放 ChatGPT Business、Enterprise 版本的应用功能，届时开发者可正式上架产品，形成 “开发 - 分发 - 变现” 的完整闭环。

轻量化模型 + Sora 2 API：多模态能力 “普惠化”

为了满足不同场景的成本需求，OpenAI 此次还发布了两款轻量化模型：

  • gpt-realtime-mini

    专注实时音视频交互，通过 WebRTC、WebSocket 连接，支持文本、音频、图像输入，输出文本与音频，价格比标准版低 70%，适合客服机器人、智能音箱等场景；

  • gpt-image-1-mini

    原生多模态图像模型，支持文本 + 图像输入、图像输出，成本降低 80%，虽生成速度较慢，但足以满足电商商品图生成、教育课件制作等需求。

而在生成式视频领域，Sora 2 API的开放标志着 “文本生视频” 正式进入工业化应用阶段。通过 5 个核心端点（创建视频、查询状态、下载、列表、删除），开发者可编程生成细节丰富的视频片段。Sora 2 提供两个版本：基础版注重速度，适合社交媒体内容、产品原型；Pro 版主打专业画质，支持 4K 分辨率，可用于电影片段、营销广告制作。

结语：AI 开发的 “效率革命” 已至

从 AgentKit 降低智能体开发门槛，到 Codex 重构编码协作流程，再到 ChatGPT 内置应用打通 “用户 - 服务” 链路，OpenAI 此次发布的产品矩阵，本质上是一场 “AI 开发效率革命”—— 它让开发者无需再聚焦 “底层技术攻坚”，而是能将精力投入 “场景创新”；让企业无需组建庞大的 AI 团队，也能快速落地智能应用。

对于行业而言，这或许意味着 “AI 创业” 将从 “技术驱动” 转向 “场景驱动”，而 OpenAI 通过开放工具、生态与用户流量，正逐步构建起一个 “开发者 - 企业 - 用户” 共赢的 AI 生态。这场盛会之后，我们或许会看到更多 “小而美” 的 AI 应用爆发，而 AI 对各行各业的渗透，也将进入 “加速度” 阶段。
