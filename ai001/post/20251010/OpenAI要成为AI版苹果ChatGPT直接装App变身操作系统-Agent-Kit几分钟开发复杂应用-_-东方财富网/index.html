OpenAI要成为“AI版苹果”：ChatGPT直接装App变身“操作系统” Agent Kit几分钟开发复杂应用 _ --知识铺

2025年10月9日
| 阅读

　　全新妙想投研助理，立即体验

　　当地时间10月6日，OpenAI开发者大会DevDay在暌违近两年后，终于在旧金山Fort Mason重磅回归。

　　一开场，OpenAI CEO阿尔特曼就投下一枚“重磅炸弹”：ChatGPT周活用户突破8亿，较2月时翻了一倍；开发者社区也从200万扩容至400万；API每分钟处理60亿个token。

　　随后，OpenAI高管相继登台，发布了一系列平台级工具和产品更新，包括Apps SDK、Agent Kit和Codex，覆盖应用生态、智能代理开发与编程辅助三大核心领域。不仅如此，模型API也迎来全面进化，包括强大的GPT-5 Pro和文生视频模型Sora 2。

　　这标志着OpenAI正在从“模型提供商”向“全栈AI生态构建者”的战略进阶，试图重新定义开发者、企业与用户的AI交互方式。

　　而在大会的压轴环节，苹果前首席设计师Jony Ive与阿尔特曼进行了一场“炉边对话”。双方确认，OpenAI正在开发不止一款硬件产品。Jony Ive称，他希望打造的硬件设备能让用户“感到快乐、充实、更加平和、更少焦虑”。

　　Apps SDK：ChatGPT直接装App，变身“操作系统”

图片来源：OpenAI开发者大会视频截图

　　OpenAI的第一个重磅发布瞄准的是其核心流量入口——周活用户已达8亿的ChatGPT。要知道，在今年2月时，这个数字还是4亿。

　　最新推出的Apps SDK为开发者提供了在ChatGPT内构建应用程序的全新路径，这也是OpenAI围绕ChatGPT打造类苹果App Store生态的关键一步。

　　Apps SDK是一套完整的开发堆栈，允许开发者在ChatGPT内部构建真实、可交互的应用程序，包括付费应用。这些应用可通过OpenAI最新发布的Agentic Commerce Protocol向用户收费，形成“开发-分发-变现”的闭环。

　　正如iOS提供操作系统，App Store负责分发变现，最后组成移动互联网生态。现在ChatGPT就是一个操作系统级的服务层，Apps SDK是开发工具，8亿用户是生态基础。

　　借助这套SDK，开发者可以连接自有数据、触发具体操作，并渲染出全交互式用户界面。换句话说，用户无需在手机、电脑或网页间切换启动应用，留在ChatGPT内即可完成所有操作，大幅降低了应用使用门槛。

　　这一功能不仅为开发者拓宽了应用分发渠道，更能为ChatGPT用户带来更丰富的体验。目前，Booking.com、Expedia、Spotify、Figma、Coursera、Zillow和Canva等公司的交互式应用程序已于10月6日上线。

　　具体来看，用户只需在ChatGPT中输入应用名称即可调用服务。例如，用户只要说：“Figma，将这张草图变成一个可行的图表”，就可以调用Figma应用；询问“Coursera，你能教我一些关于机器学习的知识吗？”，即可唤起Coursera应用。

图片来源：OpenAI开发者大会视频截图

　　在Zillow应用的现场演示中，用户用自然语言提示ChatGPT搜索特定区域、特定价格范围内的公寓，ChatGPT随即便弹出显示房源的交互式地图，用户还能通过对话进一步了解房源详情。

图片来源：OpenAI开发者大会视频截图

　　阿尔特曼说道：“（ChatGPT中的应用程序）将催生新一代交互式、自适应和个性化的应用程序。”

　　目前，Apps SDK已向开发者提供预览版。今年晚些时候，开发者将能够提交应用进行审核和发布，届时还会推出一个应用目录供用户浏览。

　　Agent Kit：告别工具碎片化，8分钟构建复杂代理

　　如果说Apps SDK重构了“人机交互”的前端界面，那么Agent Kit则聚焦后端开发效率，旨在解决AI代理开发中的工具碎片化痛点，成为打造“自主任务”执行者的核心引擎。

　　OpenAI在一篇文章中指出，此前开发AI代理需面对多重难题，例如复杂且缺乏版本控制的编排流程、需手动开发的自定义连接器、耗时的调优工作，以及发布前数周的前端开发，这些严重拖慢了开发进度。而Agent Kit作为一套“全家桶”工具集，则是为了将AI代理从原型阶段快速推向生产环境而设计。

　　Agent Kit包含三大核心组件：

　　**Agent Builder：**一套可视化画布工具，开发者无需从零开始编写代码，只需通过拖拽和连接文件搜索等功能节点，就能直观设计、测试并发布复杂的业务流程。OpenAI表示，该工具不仅适合快速迭代，还能帮助开发者清晰理解代理的工作逻辑。

　　**ChatKit：**一款可定制聊天界面，可轻松嵌入任何应用，开发者可将自有品牌元素与工作流融入其中，降低交互界面的开发成本。

　　**Evals：**一套用于评估代理性能并通过“轨迹分级”功能了解代理行为的工具。“轨迹分级”指的是分析和评估AI代理如何逐步做出决策的过程。

　　此外，Agent Kit还集成了Connector Registry功能，代理可以通过管理面板连接企业的内部工具和其他应用程序。

　　为直观展示Agent Kit的便捷性，产品经理Christina在现场完成了一场高效演示：仅用8分钟，便从零开始为静态的DevDay活动官网构建并上线了一个智能问答代理。

图片来源：OpenAI开发者大会视频截图

　　OpenAI的客户——金融科技公司Ramp也在一篇博文中印证了Agent Kit的效率提升：借助该工具，其团队仅用几个小时就构建出一个采购代理。“Agent Builder将曾经需要数月时间的复杂编排、自定义代码和手动优化工作，缩短到几个小时。”

　　不过，也有部分开发人员指出，虽然Agent Kit简化了代理构建流程，但并不意味着人人都能开发代理，它更像是补充现有流程的工具，而非替代Zapier等自动化平台。

　　目前，**Agent Builder正处于测试阶段，ChatKit和新的评估功能已正式发布。**Connector Registry正面向部分API用户以及ChatGPT Enterprise和Edu用户推出Beta版。

　　值得关注的是，Agent Kit的推出也反映了行业趋势，其他模型提供商也同样意识到了开发者工具包的重要性：谷歌今年4月推出了Agent Development Kit，将多代理系统构建简化至“不到100行代码”。运行热门代理框架AutoGen的微软，也宣布将通过新的代理框架整合代理创建功能。

　　Codex：正式版上线，重构编程工作流

　　无论是开发AI代理，还是部署ChatGPT内的应用，底层都离不开编程支持。基于此，OpenAI在本次大会上宣布，AI编程助手Codex正式从研究预览版转为正式版，进一步完善开发者工具矩阵。

　　据阿尔特曼透露，自发布以来，Codex已累计处理40万亿个Token，成为OpenAI增长最快的产品之一，其市场需求可见一斑。

　　此次Codex正式版的更新，重点聚焦企业与工程团队的深度支持，推出三项核心功能：

图片来源：OpenAI开发者大会视频截图

　　**集成Slack：**开发者可在日常使用的Slack频道或讨论帖中，直接向Codex委派任务或提问，交互方式如同与同事协作。

　　**Codex SDK：**可将支持Codex CLI的同款智能体嵌入到自己的工作流、工具和应用中，无需额外调优，即可在GPT-5-Codex上获得最佳的性能表现。

　　**全新管理工具：**借助环境控制、监测功能和分析仪表盘，ChatGPT工作区管理员如今能够在大规模管理Codex时，获得更高的可视化程度与掌控力。

图片来源：OpenAI开发者大会视频截图

　　在使用权限上，Slack集成和Codex SDK已面向ChatGPT Plus、Pro、Business、Edu和Enterprise用户开放，而新管理功能仅将对Business、Edu和Enterprise用户开放。

　　API全面升级：GPT-5 Pro与Sora 2上线

　　在工具层面更新之外，OpenAI 还带来了模型API的多项关键升级：

　　**GPT-5 Pro：**正式通过API向所有开发者开放。GPT-5 Pro是OpenAI迄今为止最强大的模型，在推理、代码质量和用户体验方面均有显著提升，尤其是针对需要分步推理、指令遵循和高风险用例准确性的复杂任务进行了优化。

　　**GPT Real-time Mini：**一款更小、更经济的语音模型，在保持高质量语音效果与表现力的同时，价格较此前版本降低70%，将大幅降低开发者构建语音应用的成本。

　　**Sora 2：**备受瞩目的文生视频模型Sora的升级版，正式开放API预览。这意味着开发者终于能将OpenAI顶级的视频生成能力集成到自有产品中。Sora 2的核心优势在于对物理世界的深刻理解：不仅能更好遵循物理定律，还具备更强的可控性，可执行更加复杂的指令，生成更逼真的效果。

图片来源：OpenAI官网

　　“芭比娃娃”品牌母公司美泰的合作案例直观展现了Sora 2对创意工作流的颠覆性影响：设计师只需从一张草图开始，借助Sora 2就能快速将其转化为可供分享、获取反馈的动态产品概念视频。

　　OpenAI“金手指”效应：Figma等多家公司股价上涨

　　OpenAI的系列发布不仅在科技圈刷屏，更迅速传导至资本市场，引发相关上市公司股价波动。多家被提及的企业，均因“应用程序将整合到ChatGPT”这一关联逻辑获得市场关注。

　　具体来看，软件公司Figma一度大涨近16%，最终收涨7.39%；HubSpot收涨2.6%，赛富时收涨2.25%。在线旅游公司也出现了短暂的股价上涨，Expedia短线一度拉涨7%，最终收盘回落至2.1%。美泰也一度拉涨超5%，但收盘时几乎持平。

　　虽然这些企业与OpenAI整合规模，远不及AMD和OpenAI披露的交易，但仍像市场展示了OpenAI的“流量带动效应”。

　　不过，这种增长与波动也引发了争议。部分观点认为，“OpenAI带来的增长和股价波动不可持续”，也有人认为，OpenAI对市场的影响力是“奇怪的新事物”。

　　有外媒指出，OpenAI的股价带动效应让人联想到去年英伟达首席执行官在一次会议上提及相关企业时所引发的市场波动。当时，黄仁勋在会议中提及工程软件制造商新思科技、Cadence Design Systems Inc.、Ansys Inc.，称这些企业将使用基于Blackwell的新型处理器增强产品AI功能，随后三家公司股价均出现上涨。

　　对于当前的市场情绪，Themis Trading的合伙人Joe Saluzzi直言：“这是一个动量市场，任何与OpenAI相关的事情都会让股票上涨，人们已经把基本面抛到了脑后。”部分投资者甚至将这种交易热情比作“互联网泡沫时代的狂欢”。

　　传奇设计师Jony Ive：联手OpenAI开发AI硬件，希望让用户“快乐、充实”

　　OpenAI本次开发者大会的压轴环节是阿尔特曼与苹果前首席设计师Jony Ive围绕“AI时代的创造哲学”展开的炉边对话。期间披露的硬件开发计划，成为整场大会的“惊喜彩蛋”。

　　据外媒，两人在对话中透露，OpenAI正在开发不止一款硬件产品，但产品用例、规格等细节仍处于保密状态。

　　Ive表示，其团队在筛选“适合集中精力发展的硬件方向”时，已构思出“15到20个真正引人注目的产品创意”。

　　值得注意的是，Ive对硬件的设计理念跳出了“效率优先”的传统框架。他承认AI具备提高生产力的潜力，但“提高效率并非是他设计这些设备的核心目标”。相反，他更看重硬件的社会效应，直言这些设备应该“让我们感到快乐、充实、更加平和、更少焦虑、更少疏离”，试图通过设计重构人与设备的关系。

　　Ive称，他想与OpenAI合作设计AI设备的原因之一，就是希望改变人们目前与日常使用设备的互动模式。

　　此前有报道称，OpenAI计划制造一种不同于手机或笔记本电脑的新型硬件。阿尔特曼在最近向OpenAI员工发布的预览中暗示，该产品将能够感知用户的周围环境和日常体验。该设备可能没有屏幕，并依赖于摄像头和麦克风的输入。

　　不过，OpenAI尚未公开硬件产品的具体推出时间，外媒此前推测计划发布时间为2026年底，但近期有消息称，该设备开发因技术问题遭遇阻碍。
