ChatGPT内嵌App！OpenAI开发者日全览，Agent工具链+应用生态+模型API多箭齐发 --知识铺

OpenAI开发者日2025，新品发布密度远超往年。

奥特曼带着一系列最新内容来了——

现在，大伙儿可以在ChatGPT中和各种“App”对话，只需在提示词中呼唤应用名即可触发调用。

比如直接在ChatGPT 5的对话框中输入“coursera_（一个提供网络公开课的App）_教我机器学习知识”，ChatGPT就能立刻连接该App并提供相关课程:

在日常对话中，ChatGPT也会根据用户的需求，主动推荐用户使用某款App：

另外，开发者构建Agent有新工具了——AgentKit。

OpenAI研究员现场演示，在8分钟内用AgentKit为其开发者日官方网站构建Agent。

AgentKit是一个Agent工具包，包含诸多模块，可以帮助开发者更高效设计工作流。

例如通过Agent Builder模块，开发者无需从代码开始，只需添加并设置一系列节点即可用可视化直观的方式快速构建Agent：

这还没完，OpenAI还宣布AI编程神器Codex正式发布，并且推出了三项新功能。

OpenAI研究员Steven Heidel在X上透露，最新发布的Agent Builder不到6周就完成了端到端构建，而Codex编写了80%的PR。

除此之外，Sora 2、GPT-5 Pro等最新模型现在也通过API调用方式向开发者和企业提供。

一波接一波的发布让网友直呼应接不暇。

下面逐一展开来看。

ChatGPT内集成APP

目前，ChatGPT内已集成的APP包括Booking.com、Canva、coursera、Expedia、Figma、Spotify、Zillow。

这些应用能自然融入ChatGPT对话，用户可通过呼唤应用名或ChatGPT适时推荐来使用，还具备交互式界面，能适配聊天语境。

首次使用某个应用时，ChatGPT会提示进行连接，用户可以知晓可能会与该应用共享哪些数据。

OpenAI预告，今年年底还会有11家合作伙伴的应用将上线。

与这些应用同步推出的，还有配套的Apps SDK。

OpenAI表示，开发者现在可借助Apps SDK着手构建和测试App，今年晚些时候将启动应用提交、审核与发布，并公布开发者如何通过应用盈利的细节。

同时，他们还会为用户推出一个专门的目录，可以浏览和搜索应用。

Apps SDK基于MCP开放标准构建，可让ChatGPT连接外部工具与数据，并在MCP基础上进行了扩展，支持开发者自定义应用界面与交互逻辑。

目前Apps SDK已开源，开发者用它构建的应用能在任何采用该标准的环境中运行。

开发者在构建App时可参考OpenAI官方指南和应用示例，然后使用ChatGPT中的开发者模式测试应用。

构建Agent的新工具

再来看AgentKit，这是一套为开发者和企业打造的、用于构建、部署和优化智能体的完整工具集。

AgentKit包含Agent Builder、Connector Registry、ChatKit等模块。

Agent Builder提供了一个可视化画布，让开发者在构建Agent的过程中可直观地理解其内部运行逻辑。

它通过拖放节点的方式来组合业务逻辑，能够连接各种工具，并支持配置自定义的安全防护规则，支持流程预览运行、内置的评估配置以及完整的版本管理。

Connector Registry则是一个集中式的管理平台，管理员能够统一管理数据和工具在各种OpenAI产品之间的连接方式和访问权限。
其中内置了丰富的预置连接器，覆盖了如Dropbox、Google Drive、Sharepoint和Microsoft Teams等常用数据存储与协作工具，还支持连接第三方MCP。

而ChatKit，让开发者能轻松地将基于聊天的智能体嵌入到自己的产品或网站中，开发者通过直接使用它，可以省去从零开始构建一个功能完善、体验良好的聊天界面所需的大量前端开发工作。

在Agent性能评估方面，OpenAI在原有Evals功能基础上，新增了四项能力：

  • 数据集： 现在可以从零开始快速构建用于评估智能体的测试数据集，并可以利用自动评分器和人工标注来持续扩充和完善这个数据集。

  • 轨迹评分： 能够对智能体的整个工作流程进行端到端的评估，并通过自动化评分来精确定位其中的缺陷和薄弱环节。

  • 自动提示词优化： 系统能够基于人工标注的结果和评分器的输出，自动生成效果更好的提示词。

  • 第三方模型支持： 可以在OpenAI的Evals平台内，直接评估来自其他供应商的模型。

目前，ChatKit和新增强的Evals评估功能已向所有开发者全面开放，Agent Builder在内部测试阶段，Connector Registry开始向部分用户开放测试，所有工具的功能都包含在标准的API模型调用费用中，无需额外付费。

代码编写工具升级

OpenAI还宣布其AI编程神器Codex现已全面可用，并推出三项新功能。

首先是Slack集成，团队成员能在Slack频道或线程里直接@Codex委派任务、提问，就像和同事互动一样，Codex会自动获取对话上下文、选好环境，用Codex云中完成任务的链接回复，之后还能合并更改、继续迭代或本地处理任务。

其次是Codex SDK，能把支持Codex CLI的智能体嵌入自己的工作流、工具和应用，借助几行代码，就能让GPT‑5-Codex发挥出色性能，而且SDK提供结构化输出和上下文管理，现在支持TypeScript，后续会有更多语言支持，同时还有新的GitHub Action，方便集成到CI/CD管道，在shell环境也能通过命令行使用Codex智能体。

还有全新管理工具，ChatGPT工作区管理员可以编辑、删除Codex云环境，给Codex CLI和IDE扩展设置更安全的本地默认值，通过分析仪表板监控使用情况和代码审查质量。

现在，Slack集成和Codex SDK对使用ChatGPT Plus、Pro、Business、Edu和Enterprise计划的开发者可用，而全新管理功能将对Business、Edu和Enterprise计划可用。
