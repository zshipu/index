有关如何使用生成式 AI 在网络浏览器中快速高效地创建优质用户界面的教程。

设计网页样式很困难。

很多时候，它们看起来很可怕——尤其是当你刚开始的时候。

体验没有捷径，但有足够好的用户界面的捷径——对我们来说幸运的是！

也许您刚开始担任 Web 开发人员，或者在训练营学习，或者您本身作为设计师并不自信。如果是这样，本文有望通过演示“生成工作流程”来帮助您快速为前端 Web 项目创建非常好的用户界面。

我们所说的“生成”是指：使用人工智能语言模型根据文本提示为我们“生成”代码。我们所说的“工作流程”是指：以一种实际有用的方式整合人工智能的使用，以支持我们可能拥有的其他目标；例如：制作一个 Killa 作品集 ⚡ 来展示您是一名优秀的 Web 开发人员 🔥。

 先决条件

  • 我们正在使用 React，这是一个前端库，用于使用 JavaScript 在 Web 浏览器中创建用户界面。了解 React 如何将页面块排列成可以在多个位置重用的“组件”，您将受益匪浅。

    不过，使用 Tailwind 并不需要了解 React — Tailwind 通常对于设计网页样式很有用。

  • 我们将使用 Node.js 的包管理工具 npm 在终端中运行命令。提供了我们将使用的命令，您不需要了解太多有关如何使用终端的知识，除了如何在计算机上访问它以及如何粘贴命令并运行它们（在正确的文件夹中！ ）

  • 如果您知道如何使用 HTML 构建网页，并且具备 CSS 样式页面的任何现有知识，那么您将从 Tailwind 中获得更多收益。 CSS（“级联样式表”）是网络浏览器用来决定如何以视觉方式“设计”网页中元素的语言。

    然而，如果这些概念对您来说是新的，那么陷入并使用生成式人工智能来支持您的学习是开始学习网络技术的好方法！

  • 为了编辑 React 项目中的文件，您将需要一个文本编辑器，例如 Visual Studio Code。在紧要关头，基于浏览器的版本应该可以：vscode.dev

那么，我为什么要关心 Tailwind CSS？

  • 在生活中，我们要以“善”为目标。如果我们点击“非常好”，那就太好了——万岁🥳！如果我们点击“确定”，那么……大多数情况下就可以了。就在网络浏览器中生成用户界面而言，Tailwind 可以帮助您更频繁、更轻松地点击“好”。它可以帮助您非常轻松且快速地点击“确定”。它还可以帮助您比其他技术更快、更轻松地实现“非常好”，甚至“优秀”😮的用户界面。我认为这值得花时间学习。
  • Tailwind 支持非常快速地进行“敏捷”原型设计。节省时间就是节省金钱💸
  • Tailwind 目前非常流行。很多网站都使用它，这意味着很多公司都使用它。您经常在网络开发职位的招聘广告上看到它。
  • Tailwind 是“可扩展的”和“可配置的”：如果需要，您可以在它的基础上构建或自定义它。
  • 人们在 Tailwind 之上构建了良好的预制 UI 组件；例如：“Tailwind UI”、“Headless UI”、“Flowbite”。

 什么是顺风？

Tailwind 是一个“开源 CSS 框架”。

为了解释 Tailwind 的本质作用，我们将解释两种设计网页样式的方法：“语义”CSS 类和“实用”CSS 类。

“语义”CSS 类是一种将多个样式规则捆绑在一起，然后将它们应用到特定 HTML 元素的方法。例如，如果我的网页有一个“介绍”部分，我可能会使用 intro 这样的类名，这样我就可以记住该类的用途，这样其他开发人员也可以看到它的用途:

<span id="9c13" data-selectable-paragraph=""><span>&lt;<span>div</span> <span>class</span>=<span>"intro"</span>&gt;</span><br>  <span>&lt;<span>h1</span>&gt;</span>Welcome to my website!<span>&lt;/<span>h1</span>&gt;</span><br>  <span>&lt;<span>p</span>&gt;</span>I hope you enjoy visiting! :)<span>&lt;/<span>p</span>&gt;</span><br><span>&lt;/<span>div</span>&gt;</span></span>

<span id="0b6a" data-selectable-paragraph=""><span>.intro</span> {<br>  <span>margin</span>: <span>2rem</span>;<br>  <span>padding</span>: <span>1rem</span>;<br>  <span>text-align</span>: center;<br>  <span>color</span>: midnightblue;<br>  <span>background-color</span>: floralwhite;<br>}</span>

我们可以非常自信地假设 CSS 类 intro 用于某种介绍性内容，并且我们可以使用 .intro 来定位网页上我们想要以某种方式设置样式的 HTML 元素在一个单独的 CSS 文件中 - 如果我们想要“分离我们的关注点”并保持我们的项目可扩展和可维护，这是一个好主意。

然而，如果我们有一个非常复杂的网页，或者我们正在构建具有大量重复组件的用户界面，我们可能会发现思考好的“语义”类名很困难；如果你不小心的话，你最终会将你的 outer-text-container-wrapper-subdivider 类与你的 primary-nav-listitem-sublist-icon-container 混淆！ 😵💫

如果您也在多个元素中重用同一个类，那么通过添加、删除或更改样式规则来更改语义 CSS 类可能会在站点的其他部分产生不可预见的后果。

“实用”CSS 类是一个非常具体的类，只做一件事。我们可以愉快地在同一个元素上组合多个实用程序类，并且浏览器在加载页面时将应用每个实用程序类的所有样式规则。

<span id="c378" data-selectable-paragraph=""><span>&lt;<span>div</span> <span>class</span>=<span>"margin-2rem padding-1rem text-center color-midnightblue background-floralwhite"</span>&gt;</span><br>  <span>&lt;<span>h1</span>&gt;</span>Welcome to my website!<span>&lt;/<span>h1</span>&gt;</span><br>  <span>&lt;<span>p</span>&gt;</span>I hope you enjoy visiting! :)<span>&lt;/<span>p</span>&gt;</span><br><span>&lt;/<span>div</span>&gt;</span></span>

<span id="e79b" data-selectable-paragraph=""><span>.margin-2rem</span> {<br>  <span>margin</span>: <span>2rem</span>;<br>}<br><br><span>.padding-1rem</span> {<br>  <span>padding</span>: <span>1rem</span>;<br>}<br><br><span>.text-center</span> {<br>  <span>text-align</span>: center;<br>}<br><br><span>.color-midnightblue</span> {<br>  <span>color</span>: midnightblue;<br>}<br><br><span>.background-floralwhite</span> {<br>  <span>background-color</span>: floralwhite;<br>}</span>

尽管对于小网页上的简单介绍性元素来说，这看起来要冗长得多，但一旦您开始创建具有许多重复组件的复杂用户界面，将规则拆分为单独的实用程序类并使用实际上会更容易、更易于管理且更易于维护他们根据具体情况进行处理。

然后，您可以在编写 HTML 的同时设置其样式。

Tailwind 本质上是一个预制的 CSS 实用程序类系统，我们可以在我们的项目中使用它。

使用 React 设置 Tailwind

相关文档可在 Tailwind 网站上找到。

让我们跳到终端并设置一个新的 React 项目。我们可以使用 Vite 为我们创建一个新的 React 应用，其中‘ my-project ’是项目的名称，也是它所在文件夹的名称：

<span id="b5d7" data-selectable-paragraph="">npm create vite@latest my-project -- --template react</span>

（末尾的 -- --template react 标志是可选的；没有它，您只需从菜单中选择您想要的项目类型（“vanilla”HTML、React、Vue 等）。）

使用 cd my-project 移至新创建的项目目录（在本例中为 my-project/ ）。

现在，在 React 项目的根目录（包含 package.json 的文件夹），我们想要使用 Node 安装一些“开发依赖项”：

<span id="8888" data-selectable-paragraph="">npm install -D tailwindcss postcss autoprefixer<br></span>

您实际上并不需要知道这些如何使用 Tailwind，但对于那些感兴趣的人来说， [tailwindcss](http://zshipu.com/t/index.html?url=https://www.npmjs.com/package/tailwindcss) 是实际的 Tailwind 依赖项， [postcss](http://zshipu.com/t/index.html?url=https://postcss.org/) 是一个用于操作和转换 CSS 文本的工具。一种可定制和模块化的方式， [autoprefixer](http://zshipu.com/t/index.html?url=https://www.npmjs.com/package/autoprefixer) 是PostCSS的一个插件，它将“供应商前缀”添加到规则中，以便支持较旧的浏览器。

（这还应该安装 Vite 运行 React 开发服务器所需的所有依赖项，但如果您对此有任何问题，请运行 _npm install_ ，npm 将检查项目的 _package.json_ 文件并安装那里指定的所有依赖项。）

要在我们的项目中“初始化”Tailwind，并在项目的根目录下创建一个 tailwind.config.js 和一个 postcss.config.js ：

<span id="60eb" data-selectable-paragraph="">npx tailwindcss init -p</span>

打开文本编辑器（如果使用 Visual Studio Code，则为 code . 打开当前目录）。要专门为 React 配置 Tailwind，我们需要将 tailwind.config.js 的内容替换为：

<span id="5e78" data-selectable-paragraph=""><br><span>export</span> <span>default</span> {<br>  <span>content</span>: [<br>    <span>"./index.html"</span>,<br>    <span>"./src/**/*.{js,ts,jsx,tsx}"</span>,<br>  ],<br>  <span>theme</span>: {<br>    <span>extend</span>: {},<br>  },<br>  <span>plugins</span>: [],<br>}</span>

.. 并将 src/index.css 的内容替换为：

<span id="2b02" data-selectable-paragraph=""><span>@tailwind</span> base;<br><span>@tailwind</span> components;<br><span>@tailwind</span> utilities;</span>

您现在应该能够通过运行以下命令在本地使用 Vite 的开发服务器为您的 React 应用程序提供服务：

<span id="6edb" data-selectable-paragraph="">npm run dev</span>

运行后检查终端输出：Vite 应提供一个链接，例如 http://localhost:5173/ ，它将在浏览器中显示您的 React 应用程序。

要检查 Tailwind 是否正常工作，您可以使用我的专利“此文本是红色的吗？”在 src/App.jsx 内测试：

 结论

公开可用的大型语言模型在使用时免费，可以使用 Tailwind CSS 进行样式轻松地为 React 应用程序生成组件。

这些组件的质量总体上很好，超过了缺乏经验的开发人员在不花费大量时间研究 CSS 和浏览器内渲染的情况下可能实现的质量。

Tailwind CSS 代表了创建“足够好”的 Web 用户界面的快速通道，而生成工作流程可以轻松地增强这一点。理论上，创建一个新的工作组件所需的时间可以缩短到几分钟。

GPT-4 很难正确识别多模式输入中的图像和水平布局，但“元提示”在一定程度上弥补了这一问题。

然而，多模式输入成功地完全准确地读取了文本——值得注意的是，GPT-4 现在可以非常轻松地从图像中解析文本，就像 Google Lens 等其他工具一样。在为语言模型构建复杂的提示时，元提示可能是“入门”的好方法：您可以测试生成的提示的输出，并相应地调整它。

文本提示和图像输入的组合可能最有效地实现生成的 UI 布局的准确性。

识别模型能力中的弱点（例如不能很好地识别图像）实际上可以通过在未来的提示中包含有关这些弱点的反馈并要求模型“特别注意”任务的这些特定方面来补救 - 实际上，启动模型本身的盲点减少了这些盲点。

使用更“命令式”而不是“声明性”的提示样式可以有效地说服语言模型首先描述它在图像模型中看到的布局，识别页面上的特定元素及其相对于彼此的布局，然后生成描述此布局的 HTML/JSX 代码。

这种方法可以与“思维链”提示相比较。
