超越 OpenAI Atlas，Agent 操作系统 FlowithOS 发布——一手实测体验总结

什么是 FlowithOS？

FlowithOS 是全球首个为 AI Agent 原生打造的桌面操作系统，而非传统的 AI 浏览器。作为 Flowith 公司发布的最新产品（2025年10月），它代表了一个根本性的转变：从"聊天机器人"到"行动型智能体"的跨越。12

简单来说，如果 Windows 是为应用程序准备的运行环境，那么 FlowithOS 就是为 AI Agent 准备的。它让 Agent 不再困在浏览器标签页或聊天框中，而是能像人类一样自动打开网页、点击按钮、填写表单，甚至执行跨平台的复杂任务。31

FlowithOS vs OpenAI Atlas：核心差异

方面 OpenAI Atlas FlowithOS
产品定位 AI 浏览器（Browser-based） Agent 操作系统（Desktop OS）
发布时间 2025年10月21日 2025年10月27-28日
运行方式 网页浏览器内集成 独立桌面应用（macOS/Windows）
执行能力 低（仅限浏览和理解） 高（系统级权限执行）
跨平台支持 限制于浏览器标签页 可跨多个网站无缝衔接
记忆能力 弱，无持久化学习 强，包含知识花园系统
适用场景 快速查询、轻量级任务 复杂工作流、多步骤自动化

性能对比：数据领先优势

FlowithOS 在权威学术测试中展现出显著优势。在 Princeton 大学的 Online-Mind2Web Benchmark 测试中（这是衡量 Agent 网页操作能力的关键指标）：

  • 整体准确率：FlowithOS 达到 95.4%，远超 OpenAI Atlas45
  • 简单任务（Easy）：96.3% 准确率
  • 中等难度任务（Medium）：97.7% 准确率
  • 高难度任务准确率：92.8%，同样领先竞品

这意味着 FlowithOS 在理解和操作复杂网页界面时，已经接近"满分"水平。4

一手实测体验：真实案例分析

实测案例：淘宝手机对比任务

一位产品经理进行了以下实测：

任务指令：“在淘宝网为我搜索并对比 OPPPO X8S+、OnePlus 13、VIVOX200，从各品牌官方旗舰店加入对比价格、配置、外观、功能亮点并总结选购指南。”

执行过程 ：3

  1. 自动打开淘宝 → 自动登录 → 搜索手机型号
  2. 依次进入各品牌官方旗舰店 → 收集产品信息
  3. 输出 Markdown 格式总结文件

关键体验指标 ：3

  • ✓ 快：登录和首轮搜索仅需 30 秒左右
  • ✓ 稳：即使跨多个页面也无中断
  • ✓ 完整：除了扫码登录确认，整个过程不到 10 分钟全程自动完成
  • ⚠ 小瑕疵：输出结果为英文格式，国内场景不够本地化

其他应用案例

FlowithOS 还可以处理以下任务：1

  • 在 B 站自动刷新、分类视频内容
  • 自动执行德州扑克游戏逻辑分析
  • 监盘并自动生成财经预测简报
  • 社交媒体账号的跨平台内容分发

FlowithOS 的三重进化架构

1. 双模态网页理解（从"看见"到"看懂"）

FlowithOS 的浏览器 Agent 不仅读取文本，还能识别按钮、输入框、图标等视觉元素，真正理解网页逻辑结构。这种能力在 Online-Mind2Web 基准测试中甚至击败了 Google Gemini 和 OpenAI Atlas。3

2. 长期记忆系统（从"任务"到"伙伴"）

知识花园（Knowledge Garden） 是 FlowithOS 的核心创新：13

  • 持续记录用户交互历史
  • 主动学习并优化行为策略
  • 能够识别用户偏好并提前筛选相关内容

根据实测，使用几天后，系统能够判断哪些信息更重要，甚至能预测用户可能感兴趣的主题。3

3. 一站式执行流程（从"构思"到"完成"）

FlowithOS 整合三个核心模块：3

  • Context Playground（创意画布）：构思和规划
  • Knowledge Garden（知识花园）：知识积累
  • Neo（Agent 创建器）：执行和实现

整个流程从构想到执行一气呵成，这意味着 AI 可以从头到尾独立完成内容运营、市场策划或技术实验。3

FlowithOS 的技术架构

执行模式6

FlowithOS 提供两种执行模式：

模式 特点 适用场景
Fast Mode 速度优先，快速交付结果 简单、直接的任务
Pro Mode 深度视觉理解、复杂界面推理、调用 Skills 和 Memories 复杂网页、需要学习的任务（推荐作为默认选项）

支持的功能6

  • 步骤追踪：实时查看 Agent 如何导航、点击、提取信息和输入数据
  • 文件管理：任务生成的所有文件和最终交付物自动整理
  • 多语言支持：支持在 GPT-4o、Claude 3.5、DeepSeek R1 等多个 AI 模型间切换
  • 技能与记忆：创建 Skills（流程步骤）和 Memories（个人偏好）供 Agent 访问
  • 预设任务库：内置专家打造的提示词库，支持自定义保存常用命令

FlowithOS 与传统 Agent 的根本区别

系统级权限 vs 沙箱限制

传统 Agent FlowithOS
仅能在沙盒内执行，无法跨页面 提供系统级执行环境，能访问网页、账号和数据
无法调用系统资源 用户授权后可进行真实系统级操作
无法处理登录验证 能自动处理登录和验证步骤，任务可继续执行

对标产品比较7

类别 普通 AI 浏览器 通用 Agent FlowithOS
定位 网页理解与总结 聊天+任务执行 Agent 操作系统
执行能力 低，仅限浏览 中，可调用插件 高，跨网页执行
系统权限 无 部分 系统级授权
记忆能力 弱 中 强，长期记忆
适合场景 阅读辅助、总结报告 单一任务执行 跨平台任务、流程自动化

定价模式10

Flowith 采用灵活的资源分配机制：

  • 免费版：每月 5 次 T-1 模型（GPT-4 级别）和 30 次 T-2 模型调用额度
  • 专业版：无限制模型调用 + 优先任务队列 + 专属算力分配
  • 企业版：支持私有化部署与定制化知识库

结论：从"助手"到"行动者"的转变

正如实测体验所示，FlowithOS 展示的是"行动的智能"，而不仅仅是"思考的智能"。它不是让 AI 更聪明，而是让 AI 真正能够做事——不需要截图、不需要人工干预，就能在网上独立完成整个工作流。3

从浏览网页到让 Agent 执行任务，从工具到操作系统，FlowithOS 让 AI 的角色第一次从"助手"变成了"行动者"。3

虽然尚处于 beta 阶段，但其性能数据（95.4% 的 Online-Mind2Web 准确率）、实测表现（不到 10 分钟完成复杂淘宝对比任务）以及系统设计（长期记忆 + 系统级权限），都标志着 Agent 操作系统时代的正式到来。也许浏览器的故事正在结束，但 Agent 的时代才刚刚开始。3 1112131415161718192021222324252627282930
