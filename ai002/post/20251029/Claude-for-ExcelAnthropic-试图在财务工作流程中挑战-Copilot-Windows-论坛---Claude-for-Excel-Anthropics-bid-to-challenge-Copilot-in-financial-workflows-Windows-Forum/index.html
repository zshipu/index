Claude for Excel：Anthropic 试图在财务工作流程中挑战 Copilot Windows 论坛 --- Claude for Excel Anthropic’s bid to challenge Copilot in financial workflows Windows Forum --知识铺

2025年10月28日
| 阅读

今天，Anthropic 悄然在企业 AI 战争中开辟了一条新战线，推出了“Claude for Excel”——一款 Excel 插件，可将其 Claude 助手直接嵌入到电子表格侧边栏中，利用实时市场连接器和预建代理技能瞄准高风险财务工作流程，并明确将公司定位为微软 Copilot 的直接竞争对手，而大多数金融专业人士每天都会使用这款应用程序。

背景：为什么 Excel 在人工智能竞赛中如此重要

Microsoft Excel 仍然是财务、会计、资金管理、销售运营和许多其他业务职能的通用语言。将人工智能嵌入 Excel 并非新鲜事；微软今年已将 Copilot 功能分层添加到该产品中，包括单元格内 =COPILOT() 函数和用于规划和执行工作流的全新多步骤代理模式 ，将电子表格从被动账本转变为主动的代理界面。微软的文档和报告明确指出，Copilot 功能定位为生产力工具，但微软也警告用户，在需要可重复数值精度的任务中应避免使用 Copilot。
Anthropic 的举动改变了格局：Anthropic 不再仅仅在云或模型层面竞争，而是在 Excel 内部与_用户接轨_ ，提供了一个替代助手，可以根据自然语言提示读取、编辑和创建工作簿，并且对财务团队至关重要的是，它能够链接到授权的市场数据和行业来源。结果，一场直接的产品级竞争，争夺企业工作流程中的一部分，而这些流程需要大量的预算，并且对错误高度敏感。

Anthropic 宣布的内容——要点

  • Claude for Excel ：一款位于侧边栏的 Excel 插件，可以通过自然语言读取、分析、修改和创建 Excel 工作簿。该插件会报告单元格级别的引用和编辑说明，以帮助追踪更改。
  • 新的市场和数据连接器 ：Anthropic 宣布与 Claude 金融服务平台建立合作伙伴关系，以在 Claude 中提供授权数据，其中包括与伦敦证券交易所集团 (LSEG) 就实时市场数据达成的已验证合作。该公司还列出了与专业供应商的连接器，例如 Aiera（收益记录和事件情报）、Chronograph（私募股权投资组合分析），以及（在 Anthropic 的生态系统材料和报告中）与穆迪 (Moody’s) 合作提供信用评级和公司数据。这些连接器通过 Anthropic 的模型上下文协议 (MCP) 和合作伙伴的 MCP 服务器交付。
  • 预建代理技能 ：开箱即用的财务技能（DCF 建模、可比公司分析、收益分析、尽职调查包、启动覆盖报告），让 Claude 可以执行多步骤项目，而不是单一提示。
  • 预览可用性 ：Excel 插件和新技能将作为 Beta 研究预览版面向 Anthropic 的付费 Max、Team 和 Enterprise 级别推出，初始用户数量有限，之后将进行更大规模的部署。Anthropic 的帮助文档确认，文件创建和高级功能将作为功能预览版面向付费计划推出。

这些细节在 Anthropic 的合作伙伴公告和同期报道中均有强调；路透社和伦敦证券交易所集团在同一天发布了伦敦证券交易所集团合作的消息。

为什么 Anthropic 的战略对财务团队很重要

财务工作流程由三个需求定义：速度、可追溯性和数据来源。Anthropic 明确地针对这三个需求展开攻势。

  • 速度：预构建的代理技能和安全环境中的代码执行旨在将数小时的手动工作（模型构建、数据提取、格式化、草稿撰写）自动化到几分钟内。供应商指出，早期部署已显著提升了生产力。
  • 可追溯性：据报道，Claude 强调_单元级引用_和审计跟踪是为了实现合规性、模型治理和可审计性——所有这些在受监管的金融环境中都至关重要。
  • 数据来源：通过模型上下文协议 (MCP) 集成授权数据源（伦敦证券交易所集团、Aiera、Chronograph 等），Anthropic 正在构建所谓的“ 数据护城河” ：高质量的、契约式的数据源，这些数据源并非在开放网络上免费提供，并且能够显著提升金融用例输出结果的事实依据。伦敦证券交易所集团的新闻稿确认了此次合作，将向 Claude 的企业客户提供授权的金融分析服务。

对于银行、保险公司、买方公司和私募股权投资公司来说，这种组合是一个可靠的产品市场契合点，这些公司更看重准确、可审计的输出，而不是泛泛的对话式回复。正因如此，Anthropic 将 Claude for Excel 定位为一款_专业_的工具，而非消费者聊天小工具。

Claude for Excel 与 Microsoft Copilot 产品的比较

两种产品都在快速发展，但它们采用了不同的架构方法。

  • Copilot 已深度集成到 Microsoft 365 和 Excel 中，并已开放由微软模型堆栈（以及 OpenAI 在多个平台上的支持）支持的单元内功能和多步骤代理模式。微软将 Copilot 定位为 Microsoft 信任边界内的租户管控功能，并设有管理员控制措施，并明确警告用户哪些情况下不应使用该功能。
  • Claude for Excel 以插件形式提供，旨在通过 MCP 连接到第三方授权数据。Anthropic 的模型托管通常运行在合作伙伴云和 MCP 合作伙伴服务器上，这不仅分散了数据访问，还引入了跨云推理路径和合同方面的考虑。伦敦证券交易所的声明和路透社的报道强调，Claude 可以通过 MCP 访问授权数据。

简而言之：微软的 Copilot 优化了内部集成和在微软合规边界内的集中治理；Anthropic 的 Claude 则通过开放协议优化了_数据丰富度_和外部供应商集成。对于客户而言，选择取决于优先级：集中式供应商控制还是广泛的授权特定领域数据。

并排快照

  • 部署面：Copilot（原生 Microsoft 365 集成）与 Anthropic（Excel 插件加 Claude 平台）。
  • 数据模型选项：在管理员控制下，Copilot 路由到 Microsoft/OpenAI（现在在选定的 Copilot 表面中路由到 Anthropic）；Claude 连接到多个许可数据 MCP 合作伙伴。
  • 可审计性和引用：两家供应商都强调出处；Anthropic 强调电子表格编辑的单元格级引用，而 Microsoft 则记录 Copilot 的功能并注意可重复性。

新的连接器生态系统实际上提供了什么？

Anthropic 的 Claude for Financial Services 合作伙伴名册（如其合作伙伴公告、伦敦证券交易所和提供商声明中所述）范围广泛且专为机构工作流程而设计。

  • 伦敦证券交易所集团 (LSEG)：通过 MCP 服务器向 Claude 企业客户授予 Workspace 和 Financial Analytics 内容的授权访问权限，从而支持时间序列数据、市场分析和 AI 就绪分类法。LSEG 表示，此举将分阶段向 Claude 企业用户推出。
  • Aiera：一家提供人工验证的收益电话会议记录和事件情报的提供商，同日宣布与 Claude for Financial Services 建立 MCP 连接器，从而实现实时投资者事件访问。
  • Chronograph：在报告中被引用为私募股权现金流和投资组合分析的提供商，被称为私人市场工作流程的连接器。
  • 穆迪和其他研究/评级提供商：包含在 Anthropic 推出和报告中，作为信用评级和研究数据的来源，但每个特定连接器的公开文档因合作伙伴公告而异。

对分析师的好处：

  • 更快地综合收益电话会议、文件和私人数据。
  • 能够利用实时市场输入和引文构建 DCF 模型。
  • 通过程序生成具有直接可追溯数据源的启动覆盖报告和推介材料。

这些连接器不是免费或附带的：它们通常需要数据提供商的许可和公司合同批准，这是 Anthropic 企业宣传的核心。

技术方法：代理技能和“私人计算机环境”

Anthropic 将功能打包为 Agent Skills—— 模块化、预构建的行为，将提示、代码执行和数据连接器链接起来，以完成多步骤任务（例如：提取文件 → 规范化财务报表 → 运行 DCF → 制作幻灯片和叙述）。Anthropic 表示，这些技能在一个_私有计算机环境_中运行，Claude 可以在沙盒中编写和执行代码，以生成输出和文件。这种设计符合大型企业的期望：独立执行、可审计的日志以及导出即用型交付成果的能力。
Anthropic 还将 Claude 中的文件创建和编辑功能作为付费层的预览版公开，使 Claude 能够直接从聊天会话中生成和修改 .xlsx、.pptx 和 .docx 文件 - Anthropic 的文档将这些功能列为 Max、Team 和 Enterprise 客户的功能预览。
这种代理能力将 Claude 从一个被动聊天机器人提升为一个主动协作者，可以在电子表格环境中执行端到端分析工作流程。

早期采用者的主张和可信度问题

Anthropic 及其合作伙伴的资料涵盖了来自大型客户的令人瞩目的早期成果。例如，Anthropic 的相关资料和合作伙伴的报道将生产力和准确性的大幅提升归功于早期的推广——其中包括引用 AIG 的一项数据，称承保审查时间“压缩了 5 倍以上”，数据准确性“从 75% 提高到 90% 以上”。这些说法在 Anthropic 的新闻资料和合作伙伴报道中反复出现。
然而，有一些重要的警告：

  • 其中一些数据来自供应商沟通或合作伙伴新闻材料，这些材料可能总结了内部试点，而不是独立的第三方审计。
  • 在公开报告中，对具体百分比改进的独立验证（例如，AIG 的“75% → 90%”准确率）有限；在撰写本文时，AIG 的直接确认或重现这些数字的监管披露在主要文件中尚不明确。读者应将供应商报告的投资回报率 (ROI) 数据视为说明性数据，直至获得独立案例研究或审计报告的验证。

Anthropic 的透明度功能（引用、审计日志）旨在缓解可信度问题，但企业社区在将关键任务流程委托给代理自动化之前，会期望可复制的、经过审计的基准。

对 IT 领导者的运营和治理影响

Anthropic 的公告提出了 IT 和合规团队必须评估的一系列具体的运营权衡。

需要评估的风险向量

  • 跨云推理和数据驻留 ：Claude 连接器和模型托管可以在第三方云或合作伙伴 MCP 服务器上运行，从而创建跨云数据路径，这些路径需要与企业数据驻留和合同义务相协调。伦敦证券交易所集团 (LSEG) 和路透社 (Reuters) 称 MCP 的推出和合作伙伴打包方案具有跨云能力。
  • 许可和数据供应商条款 ：Claude 实例中显示的许可数据需要明确的许可协议，并且可能需要支付额外费用或遵守使用限制。财务团队必须确认从许可数据生成的输出符合供应商的再分配规则。
  • 模型幻觉和可审计性 ：即使有引文和出处，法学硕士 (LLM) 也可能误解数值约束或误用公式。微软明确警告 Copilot 用户，对于需要可重复数值精度的任务，应避免使用单元格内生成函数——这一警告同样适用于任何代理电子表格功能。
  • 变更追踪和审计线索 ：引入编辑单元格的代理需要明确的策略：谁批准代理变更？“值 vs 公式”变更如何存储？所有编辑操作是否都会被记录在不可变的审计线索中？
  • 供应商和模型多样性治理 ：Copilot 生态系统本身正在向多模型编排（OpenAI、Anthropic、Microsoft）迈进。组织必须明确授权哪些模型/后端用于特定的工作负载，并以编程方式执行这些策略。

推荐的 IT 控制措施

  • 实施具有严格范围和回滚计划的试点项目（沙盒 Excel 租户、非财务生产表的抽样）。
  • 要求代理人在任何监管或报告电子表格中提出的所有变更均需人工签字。
  • 捕获完整的审计日志并要求将单元级解释与电子表格一起导出以进行合规性审查。
  • 将数据连接器视为新的服务供应商：对合同、SLA 和数据使用条款进行法律审查。
  • 量化运营成本：代理 Excel 调用、许可数据访问和模型推理都会增加增量费用——估算并监控这些成本中心。

商业影响：生产力提升与系统性风险

其吸引力是不可否认的：自动化常规建模、跨不同来源收集和规范数据以及生成初稿报告可以为初级分析师节省大量时间并大大加快决策周期。
但如果缺乏控制，速度就会产生系统性风险：

  • 嵌入到实时工作簿中的不正确的模型假设可能会通过下游报告和自动化流程产生连锁反应。
  • 在没有足够人工验证的情况下依赖代理工具可能会导致“信任衰退”的缓慢蔓延，即人类在反复接触看似合理但不正确的答案后，会不加批判地接受输出。
  • 监管利益相关者将仔细审查机构如何保持可审计性以及模型的决策逻辑如何映射到受监管的流程。

因此，实际治理不应被视为阻碍，而应被视为获取价值的必要补充。Anthropic 似乎也承认这一点——该公司强调“人在环”模型，并提供可追溯性工具——但客户在将代理系统扩展到关键任务工作流程之前，需要可演示、可复制的护栏。

这对微软-人类动力学意味着什么

微软和 Anthropic 之间的关系是对现代企业二元性的一项研究：合作与竞争可以共存。

  • 微软已将 Anthropic 模型作为可选后端集成到部分 Copilot 界面中，并将继续与其他合作伙伴合作扩展 Copilot 的代理功能。这创建了一个与模型无关的 Copilot 架构，客户可以根据不同的任务选择不同的模型。
  • Anthropic 在 Excel（微软的旗舰电子表格软件）中嵌入了一款竞争插件，这使得两家公司在屏幕空间和用户注意力方面展开了直接冲突。与此同时，在 Copilot 中支持 Anthropic 则为微软客户提供了更多选择，这对治理和采购来说极具吸引力。

对于企业来说，健康的结果是可选性加上治理：为每个工作负载使用最佳模型，但要规范每个模型的使用地点以及哪些供应商的足迹是可以接受的。

短期展望及下一步关注点

  • 采用节奏 ：关注 Anthropic 如何从最初的预览用户扩展到更广泛的企业部署，以及大型银行或资产管理公司是否会发布独立案例研究或经审计的绩效数据。早期合作伙伴媒体和 Anthropic 材料承诺分阶段推出；独立确认将成为企业采用的转折点。
  • 独立审计 ：预计第三方审计师和咨询公司将发布基准测试，比较 Copilot 和 Claude 在财务特定任务方面的表现。这些报告将具有影响力。
  • 监管审查 ：金融监管机构和内部风险职能部门在允许代理自动化触及监管报告或交易信号之前，将要求可证明的审计跟踪和模型治理。
  • 供应商经济效益 ：实时市场数据的许可成本和代理工作流程的推理费用将决定哪些企业能够大规模采用这些工具。企业应该同时考虑边际推理成本和许可数据费用。

结论

Claude for Excel 不仅仅是一款产品的发布，更是企业在争夺全球最重要电子表格内部 AI 控制权的战略升级。通过整合代理技能、文件级编辑和授权市场数据连接器，Anthropic 为金融领域提供了一种_专业的_解决方案——它既能与微软的 Copilot 直接竞争，又能充分利用互补的多供应商企业环境。
对于财务团队来说，其好处是实实在在的：节省时间、加快洞察周期，并实现重复建模工作的自动化。但同样需要注意的是：跨云推理、供应商许可、可审计性以及持续存在的 LLM 错误风险。早期采用者声称其速度提升了五倍，准确率也从 75%跃升至 90%，这令人信服，但目前许多最显著的数据都源自供应商或合作伙伴的资料，在成为大规模运营决策的依据之前，需要进行独立验证。
对于 IT 和合规领导者来说，当务之急并非挑选优胜者，而是谨慎地进行试点：全面使用各种工具，在结果至关重要的地方要求人工签字，将模型选择纳入政策，并通过独立测试验证供应商的声明。当这些治理机制到位后，代理电子表格有望成为强大的、可审计的财务加速器——但前提是机构必须坚持与其雄心壮志相匹配的纪律。
