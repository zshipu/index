Copilot 与 Claude 争夺 Excel：哪个 AI 助手会胜出？ --- Copilot vs Claude for Excel Which AI assistant wins --知识铺

2025年10月28日
| 阅读

Copilot 可直接与 Excel 无缝集成，但需要 OneDrive 自动保存，响应速度较慢，且提供的公式选项有限。Claude 响应速度更快，提供多种公式选项，并且在 Power Query M 代码方面表现出色，但需要在窗口之间复制/粘贴。对于基础用户来说，尽管 Copilot 存在局限性，但其集成性仍然很出色。对于高级用户或偏爱文件控制的用户来说，Claude 的灵活性和速度使其成为更好的选择。

我的大多数读者都知道，我尝试过很多工具，并试图利用它们来提高我和我的团队的工作流程效率。我经常探索不同的人工智能工具来提高生产力并简化我们的流程。

我之前写过关于如何在 Excel 中使用 AI 的文章，想借此机会谈谈 Copilot （内置于 Excel 中）和侧边窗口（我更喜欢使用 Claude）之间的区别。我们将讨论 Claude ，并比较和对比这两个 AI 助手如何帮助您构建电子表格和编写公式。

集成优势：Copilot 的无缝体验

Copilot 最显著的优势在于它可以直接集成到 Excel 中。当您在单元格 D5 中处理公式并需要帮助时，Copilot 已经了解您的上下文，它可以查看您的数据、理解您的列标题并引用特定的单元格区域，而无需您解释设置。Copilot 通过 Excel 中的聊天界面进行操作，允许用户以对话方式与 AI 交互，以获得快速的帮助和指导。

基本用户示例

假设您在 AC 列（销售人员、地区、销售额）中有销售数据，并且想要计算“西部”地区所有销售额的总和。使用 Copilot，您只需输入“计算西部地区所有销售额的总和”，它就会生成“=SUMIF(B:B,“West”,C:C)”函数，同时理解您的确切列结构。

高级用户示例：

对于涉及多个条件的更复杂场景，Copilot 可以生成：`=SUMIFS(C:C,B:B,”West”,A:A,”John”)` 并自动调整到您的特定数据布局。

然而，这种整合也带来了重大的代价，令许多用户感到沮丧。

自动保存的困境：为什么 Copilot 会强迫你这么做

Copilot 要求您启用自动保存功能，这意味着您的文件必须保存到 One Drive 或 SharePoint。这不仅仅是一个小小的不便，它从根本上改变了您使用 Excel 文件的方式。

启用自动保存功能后，您所做的所有更改都会立即保存到云端。虽然这可以防止崩溃导致数据丢失，但也意味着您无法轻松地尝试公式，因为这可能会覆盖您的工作。如果您犯了错误或想尝试其他方法，您可能需要依赖 Excel 的版本历史记录或手动重新加载旧文件。

对于像我这样更喜欢控制文件保存时间和方式的用户（在开发过程中创建多个版本或拥有刻意保存点的安全网）来说，这个自动保存要求是一个交易破坏者。

速度与选择：克劳德的闪光点

Claude 的响应时间始终优于 Copilot。Copilot 可能需要几秒钟才能生成响应，而 Claude 通常能在 1-2 秒内给出答案。Claude 通常能提供简洁的答案以及多种替代方案。

更重要的是，Claude 针对同一个问题提供了多种公式选项，并且其公式建议以准确的答案而闻名，这有助于用户信任这些建议。

示例比较：

问题： “如何从另一张表中查找值？”

副驾驶回应：

`=VLOOKUP(A2,Sheet2!A:B,2,FALSE)`

克劳德回复：

`=VLOOKUP(A2,Sheet2!A:B,2,FALSE)` – 传统 VLOOKUP

`=INDEX(Sheet2!B:B,MATCH(A2,Sheet2!A:A,0))` – 更灵活的 INDEX/MATCH

`=XLOOKUP(A2,Sheet2!A:A,Sheet2!B:B)` – 现代 XLOOKUP（如果可用）

Claude 还提供了效率建议，指出虽然 XLOOKUP 很方便，但 INDEX/MATCH 对于大型数据集使用的系统资源较少。

有效使用 Claude 进行数据分析的最佳实践

为了充分利用 Claude for Excel 公式，请遵循以下做法：

  • 指定目标单元格。 不要问“如何对这些值求和”，而要说“我想对 B 列等于‘West’的 C 列中的值求和，并将结果放入单元格 D10”。
  • 提供现有公式上下文。 如果您要基于现有公式进行构建，请与 Claude 分享。例如：“我有这个公式：=SUMIF(B:B,‘West’,C:C)，但我需要为 2024 年 1 月 1 日之后的日期添加另一个条件。”
  • 请注明您的 Excel 版本。Claude 可以根据您是否可以使用 XLOOKUP 等较新的函数或是否需要继续使用旧版本来定制回复。Claude 的上下文感知写作功能可确保其建议与您的具体场景紧密相关。

高级示例 – Power Query M 代码：

Claude 在 Power Query 开发方面明显优于 Copilot。Copilot 不存在于 Power Query 编辑器中，因此无法提供 M 代码辅助。Claude 可以为 Power Query 任务生成代码，使用户能够根据自然语言提示快速创建 M 代码。

场景： 您需要将日期为文本格式“YYYY-MM-DD”的表转换为实际日期值。

在提供代码示例之后，Claude 还充当代码解释器，帮助用户理解和调试 M 代码。

Claude M 代码解决方案：


= Table.TransformColumns(

&nbsp; &nbsp; PreviousStep,

Claude 还可以解释代码、提出优化建议，并为复杂的数据转换提供替代方案——而这些是 Copilot 在 Power Query 环境中无法做到的。Claude 生成的内容不仅包含代码，还包含详细的解释和替代方案，从而支持用户进行全面的 AI 驱动内容创作。

The multi-monitor reality

The copying and pasting between Claude and Excel is often cited as a disadvantage, but in today’s multi-monitor setups, this is less of an issue. Having Claude open in a side window while working in Excel actually provides some benefits. Using multiple AI tools in parallel on separate screens can further enhance productivity and enable you to leverage specialized features for coding, data analysis, and task management:

  • You can reference Claude’s explanations while implementing formulas
  • Multiple formula options remain visible for comparison
  • You can easily copy different variations to test in your spreadsheet

对于单屏用户来说，这种工作流程无疑更加繁琐，尽管存在其他限制，但 Copilot 的集成仍然更有价值。

实际示例

基本用户场景：条件格式

任务： 突出显示 D 列中值大于 1000 的单元格。

Copilot： 提供条件格式规则，但需要您浏览 Excel 的界面。

Claude： 提供公式 =D1>1000，并准确解释在条件格式对话框中的具体应用位置，此外还提供 =D1>AVERAGE(D:D) 等变体，用于突出显示高于平均值的值。Claude 和 Copilot 都可以帮助用户编写复杂的 Excel 公式代码，使高级任务更容易上手。

高级用户场景：动态数组公式

任务： 创建购买金额超过 500 美元的客户的唯一列表。

副驾驶： =UNIQUE(FILTER(A:A,B:B>500))

克劳德： 提供相同的公式，但也提供：

  • =SORT(UNIQUE(FILTER(A:A,B:B>500))) – 排序版本
  • 使用数组公式替代旧版 Excel
  • 动态数组行为和溢出范围的解释

结论：选择你的人工智能助手

对于主要使用简单公式且不介意 OneDrive 要求的基本 Excel 用户来说，尽管响应时间较慢，但 Copilot 的无缝集成使其成为更方便的选择。

对于需要灵活性、速度和 Power Query 辅助的高级用户 ，或者希望掌控文件保存工作流程的用户，Claude 无疑是更佳的选择。Claude 的速度、丰富的选项以及更广泛的 Excel 生态系统支持弥补了复制粘贴带来的一些不便。

理想的情况可能是同时使用两者：使用 Copilot 获得快速、上下文感知的帮助，使用 Claude 获得更复杂的问题解决和 Power Query 开发。
