一夜之间，Claude猛转向！Coding转向白领，Anthropic内部负责人自曝设计思路，开发只是小切片，目标是所有复杂领域 --知识铺

2025年10月29日
| 阅读

编辑 | 云昭

好家伙，一觉醒来，Claude 要从Coding的程序员转向办公白领了！

就在几个小时前，Anthropic 的一则新发布“Claude for Excel”直接引爆了Hackernews上的用户，目前评论已经高达348条。

从coding赛道拓展到office，虽然说已经是业界共识，但Anthropic这一突然的行动，还是让业界为之一震。

从官方放出的界面和资料来看，看来又是一项堪比ClaudeCode的实用神器。

整体感觉，跟两年前的 Excel Copilot 产品还是有很多不同之处的。

有了它，现在可以直接在 Microsoft Excel 中使用 Claude。它可以读取和修改电子表格，解释每一个变化；从头开始构建模型；修复损坏的公式。 

具体来看看，它的几项高亮的技能：

一、几秒内获得任意单元格的答案
即时浏览复杂模型。你可以向 Claude 询问特定公式、整张工作表，或跨多个标签页的计算流程。每个解释都附带单元格级引用，便于验证逻辑。

二、测试不同情景，不破坏原公式
在不破坏依赖关系的前提下，更新整个模型中的假设条件。快速测试不同情景——Claude 会高亮所有变更，并附上解释，确保透明可追溯。

三、调试并修复错误
几秒内追踪 #REF!、#VALUE! 或循环引用错误的根源。Claude 会解释问题原因及修复方式，不影响模型的其他部分。

四、构建模型或填充现有模板
可根据你的需求从零创建财务模型，或在保留公式与结构的前提下，用最新数据填充现有模板。

但现在还处于邀测阶段，Max、Teams 和 Enterprise 用户现在即可享受 Beta 版访问权限。有网友爆料道，waitlist 仅1000个体验名额。

可以让子弹飞一会，到时候小编再为大家呈上一手测评。

Cladue下一个野望：金融

你以为只有excel方面的动作？不，这还没完。

Anthropic索性把他的下一个商业版图直接抖出来了：Finance。

我们正在扩展Claude金融服务功能，包括Excel 插件、实时数据和市场分析的新连接器以及预建的Agent功能，包括现金流模型和启动覆盖报告。

为什么从Coding转向金融？

众所周知，Claude是模型厂商，从OpenAI离开后，一直以构建“安全、可信任”的模型为己任，但后来慢慢走向了编程领域的商业化道路。

那这次突然的赛道转向，又是为何呢？

在今天同步放出的官方录制的视频中，对外解释了这一问题。

Anthropic 金融服务部门应用 AI 工程团队负责人 Alexander Brick表示，此举是为了把研究成果转化为复杂行业中安全落地的产品。

Anthropic 最初是一家专注 AI 安全研究的机构。如今，我们正把研究成果转化为可在复杂行业中安全落地的产品。我们的目标，是构建能够安全解决世界最复杂问题的模型。

以编程为例，全球只有约 0.5% 的人口是软件工程师，而企业运行的逻辑几乎都建立在代码之上。这意味着 Claude 不仅能理解复杂系统的运作，还能清晰地展示它的思考与推理过程。

而编程还只是复杂问题中的一个小切片，世界上还有大量的难题分布在各个领域，我们的目标是让模型能逐步进入并解决这些问题。

金融领域同样如此——这里的问题复杂、监管严格、必须具备可验证性与精确性。而这正是 Claude 在金融行业中展现优势的原因。

Claude for Financial Services 产品负责人 Nick Lynn 还进一步分享了为什么金融领域特别适合大模型去赋能。

如今的金融分析师常常花大量时间去打磨细节，比如在 PowerPoint 或 Excel 模型中精确到像素的层级，不能出现任何错误。有趣的是，我们现在进入了一个新阶段——模型也能做到类似的事情，只不过它是通过结构化逻辑的能力来实现的。

这正是我们发现语言模型特别擅长的领域，也是我们重点训练它们的方向。而这种逻辑推理与结构化能力，正在被抽象并迁移到越来越多的领域——比如创建结构精密的电子表格、制作 PowerPoint 等。这种扩散的速度和范围，对我来说相当令人震撼。

设计思路公开

Claude for 金融，跟其他产品有何不同？

Alexander 常用三个动词来概括 Claude for Finance 的设计思路：检索（Retrieve）、分析（Analyze）、创造（Create）。

先说“检索”。市场上许多研究型智能体已经相当成熟，大语言模型在处理大规模数据、提取洞察方面非常出色，阅读速度是人类的几千倍。但Anthropic在金融领域更关注的是，如何让 Claude 能直接连接分析师的核心数据源。因为在金融业，能否比竞争对手更快地挖掘洞察，就是核心竞争力。

其次是“分析”。信息检索之后，更关键的是能否在规模化环境下进行分析——无论是通过代码还是电子表格。金融模型并不只是“漂亮的 Excel 表”，它代表的是分析师对未来走势与企业估值的判断。所以我们希望 Claude 能真正理解核心金融概念，并能像专业分析师一样操作 Excel、运行财务模型。

第三是“创造”。金融从业者的工作，最终都是为了与他人共享成果。无论是 Excel、PPT 还是 Word 文档，输出必须是“客户可用、会议可呈现”的。我们希望 Claude 能一体化完成这一过程，让整个工作流从检索、分析到生成都能自动衔接，形成真正的智能系统。

提到创造，小编多说一嘴，Claude这次转向早有端倪。一个月前，Anthropic 就推出了文件创建的功能。可以说，这次是真的在金融产品闭环了。

Claude产品内部，究竟是怎么工作的？

很明显，与Coding不同，金融场景对于专业背景要求极高，而且数据相对不容易获取。Claude内部如何开发出来这样一款产品的呢？

Alexander也分享了内部的做法。 

首先是模型层。“Anthropic 本质上是一家研究机构，我们的目标是让 Claude 成为最适合金融行业的 AI 模型。”

金融领域有独特的挑战——代码可以每天被测试，但在我们团队内部，真正有投行经验的人并不多。因此我们非常重视与早期客户的共创，比如 BCI、Perella Weinberg、NBIM 等，让他们告诉我们最关键的应用场景、他们眼中“好的效果”是什么，同时帮助我们识别差距，反馈回研究过程。

其次是产品层。Agent 能力就是内部团队编写的代码，用来帮助用户与模型交互。

“我们推出了 Deep Research 等功能，现在正重点投入，让 Claude 能嵌入所有核心工作界面——不仅是 Claude Enterprise Cloud AI，还包括浏览器插件、Excel、Chrome 等，让分析师能在日常工具中直接使用 Claude。”

最后，平台层。Claude产品团队希望构建一个高度灵活、易于定制和部署的平台。

我们正与 S&P、FactSet、PitchBook 等行业伙伴合作，开发深度集成，让 Claude 的金融 Agent 能更强大。

网友：Claude现在还没到那个水平

从 Hacker News 的几条高赞评论中，可以看出专业用户对 Claude Code 的态度——既谨慎，又怀抱期待。

“我觉得像 Claude 这样的工具现在还没到那个水平。”

一位从业者 jbs789 认为，把 Claude Code 交到未经训练的用户手里“会是灾难性的”。这类模型要真正发挥作用，前提是使用者必须理解背后的逻辑与数据细节。

他提到，目前 Claude Code 的合理用途更多是作为“检查器”或“验证器”，帮助发现潜在错误或辅助建模。但若不经过人工复核，他不会信任任何模型生成的财务结果。

这其实点出了一个关键现实：AI 可以生成，但人必须校对。

另一位评论者 sally_glance 则表示，AI 生成 Excel 表格或脚本完全可行，但前提是——必须有完整的文档，明确每个隐含逻辑和业务关系。否则，大模型只会在“黑箱”中生成错误的数字。这类模型更适合做自动化的“起点”，比如创建带公式和宏的模板，而非直接出具计算结果。

更值得注意的是，网友“sothatsit” 则把 Claude 与 GPT-5 作了直接比较。他认为 GPT-5 Pro 在代码审查上的严谨度已经超越人类，即便是非常仔细的程序员。

他推测，这种能力迁移到 Excel 等领域后，可能带来显著质量提升——因为 Excel 表格的错误率往往比代码更高。

但目前，Claude 在代码审查方面仍落后于 GPT-5，因此他对 Claude Sonnet 4.5 在金融领域的表现持保留态度。

不过我已经开始信任 GPT-5 Pro —— 它在发现软件 bug 这件事上，往往比我还细致，即使我已经非常小心。

如果照这个趋势来看，用这些工具来帮忙审查 Excel 表格，也可能显著提升质量。毕竟，Excel 表格的错误率往往比软件还高。

话虽如此，Claude 在代码审查能力上仍明显落后于 GPT-5，所以我对 Sonnet 4.5 在这个新领域的表现也不太确定。

我觉得 OpenAI 可能会做得更好。

写在最后：Anthropic 求深，OpenAI求广

一个有意思的现象是，全球Top2的AI独角兽，OpenAI 与 Anthropic ，时至今日，正好走在两条相反的道路上：一个toC不断烧钱要规模，一个toB，从开发场景扩展到其他垂直领域。

OpenAI 依靠规模取胜，锁定大众市场，通过订阅（未来或广告）实现变现。这一策略带来极高的曝光度，但盈利路径仍不明朗——毕竟，大多数用户仍使用免费版本，而广告模式能否在不损害用户信任的前提下融入对话体验，仍是未知数。

Anthropic 则追求“深度”而非“广度”。公司面向企业构建专业化工具，而非服务个体消费者。其主要收入来自为企业提供可靠、高质量的工作型 AI 系统。这样的定位让 Anthropic 能在相对较少的用户基础上获得更高的单客收益，同时也更容易实现稳定现金流——因为企业能直接量化 AI 自动化带来的成本节约与效率提升。

这两种模式可以说概括了现在市面上的AI公司的众生相。泡沫、非泡沫在这两家公司上折射得淋漓极致。

且让子弹，快乐地飞吧！
