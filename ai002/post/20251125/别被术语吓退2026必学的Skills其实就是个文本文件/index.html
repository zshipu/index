📌 2026年最火的AI新概念

从2025年下半年到2026年，AI圈的新热点是Skills，这个比MCP的影响力似乎还要大一点。

我估计很多人一听就开始犯难了，这又是什么东西，新消息层出不穷，跟不起了呀。

其他的你可以这么认为，但是Skills这个东西你还是要停留一下。

它特别有用，并且它特别简单！

重要的话说三遍：

Skills，人人都能学会！

Skills，人人都能学会！

Skills，人人都能学会！

这不是夸张。因为它的物理实体就是一个md文件。这md文件又是啥？其实就像word可以被叫作doc或者docx文件一样，md文件就是一个文本文件。

文本文件，任何人都能用记事本、VS Code或任何文本编辑器打开和编辑它。

我们随便打开一个skills：

真的就是一个文本文件，里面记录了提示词。复杂的可能还会有一些Python文件啥的。但核心仍然是那个.md文件。

💡 为什么一个文本文件能掀起革命？

那你可能会问了：啊？大家天天说的Skills，就是一个文本文件？那怎么会这么影响力大呢？

因为这个提示词文件，被巧妙地用在了一个系统中。

这个系统就是Claude Code。

到这里，我们又得说，没有Claude账号也可以玩Claude Code。

使用国产大模型也可以驱动Claude Code。

我们说回Skills。

想想你的日常，是不是收集了一堆提示词，用的时候就复制粘贴，或者你做了很多的智能体，GPTs，Gem什么的。但更多数时候，直接就忘记了。对吧？

因为你的保存行为，并没有把提示词纳入到系统中。

而Skills保存你的提示，是把它弄在Claude Code之中。

Claude Code这个东西，可以动态地加载Skills。

这就是奥妙之所在了。

你的Skills只要存到Claude Code里，Claude Code就会当个事儿办，会在任何合适的时候提醒你用它。

首先，Claude Code能“看到”你有哪些Skills。

然后，它还可以主动建议：“这个任务是否需要用XX Skill？”

最妙的就是，所有Skills都可以存在Claude Code系统里，随时调用。

🖥️ Claude Code：给LLM一台电脑

这还没完，更关键的是Claude Code你可以认为它是一个环境。它的作用就是让大模型可以操作电脑。

那让大模型操作电脑有什么好处呢？

好处在于，大模型什么都懂，理解力max，所以它操作电脑比人操作电脑要厉害得多。

⚡ 效率对比：1小时 vs 1分钟

这么一对比，你就明白了，提示词本来只能问答，但是在Claude Code加持之下，你的提示词就不限于回答问题了，编辑文档，调用其他服务，几乎所有你能在电脑上做的事情，Claude Code配合Skills都可以做。

并且还做得飞快。

比如，我们分析100个CSV文件并生成报告，如果是人类操作：

  1. 打开 Excel（2分钟）

  2. 逐个导入文件（10分钟）

  3. 手动创建图表（15分钟）

  4. 写报告（30分钟） 总计：~1小时

那如果是换成Claude Code + Skills：

  1. 调用“数据分析 Skill”（1秒）

  2. 批量处理所有文件（30秒）

  3. 自动生成可视化（10秒）

  4. 生成标准化报告（20秒） 总计：~1分钟

所以，这就是“给大模型一台电脑”的意义。

而且像例子里说的数据分析这个Skill，下次你直接在Claude Code里喊一嗓子，它就直接调用了，不用你再去找提示词了。

🎨 Skills能做什么？超乎你想象

程序员用的Skills就不说，普通人可以把编辑文档，生成财务报表，制作精美ppt，任务完成发送通知给邮箱，批量处理图片，爬取网络动态追踪信息，周报自动生成……等等都可以做成定制化的Skills。

如此一来，你就知道为什么Skills这么火爆了吧。当你把任何工作流或者功能，都写成md文档以后，AI一看就什么都知道了，它可以直接操作你的电脑，帮你完成你的工作。

如果我们把AI的发展比作人类的成长：

  • ChatGPT时代：AI是个聪明的学生，你问它答。

  • GPTs/智能体时代：AI是个专业顾问，有了特定领域知识。

  • Skills时代：AI是个真正的同事，不仅懂还能动手干活。

这个跨越的关键就在于：AI从“回答问题”进化到了“执行任务”。

✨ 这是一场工作方式的革命

现在你明白了吗？

Skills 不是一个功能，是一场工作方式的革命：

  • ❌ 过去：人适应电脑（学习各种软件操作）

  • ✅ 现在：电脑适应人（AI理解你的意图自动执行）

  • ❌ 过去：重复劳动消耗生命（每天做同样的事）

  • ✅ 现在：经验固化成Skills（AI代替你重复）

  • ❌ 过去：工具分散各处（要记住100个软件怎么用）

  • ✅ 现在：一个AI调度所有工具（你只需要说人话）

所以，与其问：Skills能干什么。

不如问：你想让它帮你干什么？

只要你能在电脑上做的事，都可以写成Skill让AI帮你做。

这就是2026年的工作方式。

💪 如何创建和使用Skills？

那么问题来了，Skills怎么制作？写个md文档就可以吗？写好了怎么用？

别急，答案比你想的还简单。你连md文档都不用自己写！

不过，这就必须回到Claude Code了。安装好Claude Code以后，你直接可以口语告诉它要新建一个Skills，就可以开始新建了。全程自然语言交流即可。

所以，你首先得安装Claude code。

安装Claude Code的前提条件：

  • 安装 Node.js 18 或更新版本环境。搞不定环境配置的，直接把报错扔给DeepSeek，它会手把手教你装好Node.js。

  • Windows 用户还需安装 Git for Windows

👌 顺带说一下Claude Code

除了上面可玩性极高的Skills，Claude Code对普通人来说，用处也极广。

比如，我最近想在已经安装的app里寻找一个app，但是我怎么都想不起来名字，在过去，我会挨个看过去寻找。但是有Cluade Code以后，我直接告诉它，我有个app想不起来名字了，它是干啥干啥的……

claude code根据你的描述，迅速扫描了我所有的app，告诉我可能是其中三个，我仔细一看，想要的app赫然在列。

一个月前，我有个三万条数据的表格要处理，在过去我肯定会说自己不会，但是有了Claude Code，我就告诉它目的是什么，让它帮我操作。只见它解决了一群环境设置问题，还有各种报错，帮我写了一个Python，在十几分钟之内，帮我生成了一个新表格。利用数据查询，我对比了几十条，发现全部正确。

还有一次，我让它帮我抠图，抠图后它建议我弄一个Skills，我都惊呆了。所以有一些抠图任务，我都会试试。

⚠️ 必看！新手避坑指南

重要提示：Skills 的保存位置很关键！

❌ 错误做法：让 Claude Code 随便建在默认位置

  • 它会放在 plugins/marketplaces/ 文件夹

  • 一旦更新，你的 Skills 可能被清理掉

✅ 正确做法：明确指定保存位置

  • 创建时告诉它：“把这个 Skill 保存到 skills 文件夹”

  • 使用时说：“用 skills 文件夹里的 XX 来处理这个任务”

这样你的 Skills 就永久保存，再也不怕丢失了！
