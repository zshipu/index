别只用 Cursor 写代码：手把手教你配置 Skills 技能库，解锁“完全体”工作流 --知识铺

2026年1月8日
| 阅读

⚠️ 前排提示：本文涉及命令行操作与配置文件修改，建议收藏后在PC端实操。

大多数人对 Cursor、Trae 这类 AI 编辑器的使用，还停留在 Cmd+K 生成代码的阶段。

这就像买了一辆法拉利，却只在小区里送快递。

最近，Anthropic 官方发布了一套 Skills（技能库），配合开源项目 openskills，我们可以直接把这些能力“插”进 Cursor 里。

装上它，你的 Cursor 就不再只是一个程序员，而是一个能画 PPT、能分析文档、能自我进化的超级 Agent。

今天，拆解这套工作流，把你的编辑器变成“军火库”。

01 为什么你需要 Skills？

在 Chat 框里写 Prompt 是“软约束”，AI 可能会听，也可能会幻觉。

Skills 是“硬约束”。

它本质上是一套标准化的 SOP（标准作业程序）和工具包。当你调用 Skill 时，AI 不再是瞎猜，而是严格按照你定义的步骤去执行任务——查文件、跑脚本、生成报表。

确定性，是工程化的第一原则。

02 极简部署指南 (3步走)

要在 Cursor/Trae 中使用 Skills，我们需要一个中间件：OpenSkills。

第一步：安装“连接器”

打开你的终端（Terminal），输入以下命令安装 openskills 工具：

<section></section><section><p><span leaf="">npm install -g openskills<span leaf=""> </p></section>

第二步：进货“技能包”

安装完成后，我们需要下载技能。Anthropic 官方提供了一套高质量的 Skills（包含 PPT 生成、文档分析等）。

在你的项目根目录下运行：

<section></section><section><p><span leaf=""># 安装到当前项目<span leaf=""> </p><p><span leaf="">openskills install anthropics/skills<span leaf=""> </p></section>

运行后，你会看到一个交互界面，按 Space 键选择你想要的技能。建议全选，特别是 pptx 和 skill-creator。

安装成功后，你的文件列表里会出现一个 .claude/skills 文件夹。这就是你的“军火库”。

第三步：注入灵魂 (关键！)

这是最容易被忽略的一步。

光下载没用，你得让 Cursor 的 AI “看见” 这些技能。我们需要生成一个 AGENTS.md 文件，作为 AI 的行动指南。

在根目录运行：

<section></section><section><p><span leaf="">openskills <span leaf="">sync<span leaf=""> </p></section>

系统会自动创建一个 AGENTS.md。这个文件告诉 AI：“嘿，我有这些新本事（Skills），当用户问相关问题时，请查阅这里。”

03 实战：它能干什么？

部署完毕，现在的 Cursor 已经不是昨天的 Cursor 了。

场景一：一句话生成技术 PPT 📊

以前写汇报 PPT：找模板、填内容、调格式，耗时 2 小时。
现在，直接在 Cursor 对话框输入：

“调用 pptx skills，根据最近的 git commit 记录，整理一份最近改动点的汇总 PPT。”

Cursor 的思考路径：

  1. 读取 AGENTS.md 发现有 pptx 技能。

  2. 自动抓取 git log。

  3. 调用 Python 脚本生成 .pptx 文件。

结果？一份结构清晰、配色专业的 PPT 直接出现在你的目录里。你只需要微调即可。

场景二：让 AI 自己写 Skill 🛠️

Anthropic 提供了一个 skill-creator。你可以让 AI 根据你的需求，自己写一个新的 Skill。

需求示例：
“我经常需要分析 /fixes 目录下的修改记录。请调用 skill-creator，帮我做一个自动生成‘改动报表’的 Skill。”

AI 的操作：
它会分析你的需求，自动编写 Python 脚本，定义输入输出，并将其注册到 AGENTS.md 中。

从此，你拥有了一个专属的“改动分析员”。

04 技术复盘与思考

这套工作流的本质是什么？

是 RAG (检索增强生成) + Tool Use (工具调用) 的本地化实践。

  • 以前：我们需要把所有上下文塞给 AI，祈祷它记住。

  • 现在：我们将能力封装成 Skill（代码+文档）。AI 变成了“调度员”，它不需要知道怎么画 PPT 的每一个像素，它只需要知道何时调用那个画 PPT 的工具。

✅ 节省 Token：不需要每次都把规则重复一遍。
✅ 高确定性：代码逻辑保证了输出格式的稳定。
✅ 可复用性：一次编写，所有项目通用。

写在最后

AI 编程工具的下半场，不再是比拼谁的代码补全更快。

而是比拼谁能更好地组织 Agent。

当你开始编写 AGENTS.md，当你开始积累自己的 Skills 库时，你就不再是一个单纯的 Coder，你是一个 AI 团队的 Leader。

建议现在就打开 Cursor，试着装上你的第一个 Skill。
