Skills编写无从下手？Claude Code Skills制作的六步框架 --知识铺

2026年1月8日
| 阅读

Brandon Hancock 最近在视频中分享他的Claude Code Skills Master Framework六步框架，教会你如何在六步完成Skills的编写。（视频地址见文章结尾）

其实这里面步骤有很多与我们在AI编程高效开发指南专栏里面讲的内容不谋而合，不过Brandon的更成体系。

而且看完之后，我觉得它不仅仅只是作为Skills的编写，而是可以作为所有AI编程以及Agent编排的参考步骤。

归根到底，它是一套系统化思维，指导如何从一次性的手动操作出发，逐步提炼逻辑、标准化流程，最终封装为可复用、高可靠的 AI 技能（Skill）。

步骤 1｜手动执行优先

自动化的第一原则是，如果你不能手动完美地完成它，你就无法自动化它。 

在这一步，你的目标不是直接写Skill，而是演练你的操作过程。

说得更简单一些， 即使任务看起来很简单，也要亲自、完整地手动操作一遍。不要依赖想象，因为细节当中藏着魔鬼。

整个过程当中详细记录你需要输入及任务完成后的具体成果的输出是什么。

这是后续标准化的基础。

特别留意哪些步骤容易卡顿、歧义或出错。这些就是未来自动化脚本中最需要做 防呆设计的地方。

步骤 2｜分析与反馈

在手动执行后，不要急着写内容。

第二步是复盘，审视你的手动执行过程和结果。

他的建议是建立一张简单的表格来记录分析过程，它可以作为编写我们后面Skill.md的依据。

步骤 3｜系统化为技能

这个步骤的关键是模块化拆分。

大任务拆小，独立治理， 让一个脚本只做一件事，同时理清模块间的调用关系，避免循环依赖。

步骤 4｜训练新技能

写好了Skill的脚本代码，不代表 AI 会用，步骤 4 是编写说明书，教会 AI 在何时、如何调用这个技能。

语气要指令性强，告诉 AI “当用户询问 X 时，使用此工具”。

至少提供 3 类示例：正常情况、缺少参数的情况、参数错误的情况。告诉 AI 在这些情况下该如何反应。

在沙箱或测试环境中，模拟用户提问，检查 AI 是否正确识别意图并发起了正确的函数调用。

步骤 5｜迭代优化

步骤 5 是一个循环迭代Loop的过程，目的是提升技能的鲁棒性

步骤 6｜推广落地

最后一步，是将你的个人技能转化为团队资产。
