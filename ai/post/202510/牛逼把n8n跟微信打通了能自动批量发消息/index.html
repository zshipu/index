版本：v1.2.27｜更新时间：2025-09-28  插件：n8n-nodes-weixin-wechat｜节点：WeChat Send  适用人群：希望用 n8n 自动发送微信消息的同学（完全小白也能上手）

前段时间结合claude code开发了个n8n插件，主要的claude code使用方法可以参考上一篇文章：用Claude Code AI编程，三天做出一个N8N微信插件

这篇文章主要是想给大家分享怎么用n8n微信插件，因为最近突然又有好多人来问我。

先给大家看看插件效果吧，他是通过模拟人工操作控制鼠标。

一、这篇文章帮你解决什么

一）先跑通：用“西羊石n8n微信插件”完成第一条消息发送

二）怎么选：个人微信 vs 企业微信，哪个更适合你的场景

三）怎么填：节点里每种消息类型的必填字段一目了然

四）怎么稳：批量发送的控频参数与常见故障的对症排查

二、功能对照 & 选型建议

目前我用的比较多的还是在个人微信场景，尤其是批量发送，非常给力，目前账号没有被封过，大家害怕有风险也可以先尝试用小号。

选择建议：

  • 小规模私域触达、发资料、偶发群发 → 个人微信

  • 项目播报/日报、监控告警、格式化文档 → 企业微信

三、先跑通最小闭环（建议照抄）

目标：把一条文本发到文件传输助手，确认“链路通”。

一）准备环境（个人微信通道）

  • Windows 10+ / Windows Server 2016+

  • 微信 PC 版 3.9.8–3.9.12（4.0+ 不兼容此方案，以实际兼容性为准）

  • 电脑保持不休眠（因为程序得看到页面和控制鼠标）

右键单击我的电脑，打开属性面板，查看windows系统。

微信版本查看是登录电脑版微信，点击左下角，打开设置。

在最下面关于微信能查看到版本信息。

二）获取 API 密钥

  1. 关注公众号「西羊石AI视频」

  2. 发送关键词「API」，不是在视频号后台。

  3. 复制形如 httpxxx 的密钥并注意不要有前后空格。

三）在 n****8n 安装插件  设置 → 社区节点 → 安装 n8n-nodes-weixin-wechat

四）启动个人微信 PC 服务

打开仓库并下载代码：https://github.com/Standed/n8n-nodes-weixin-wechat

进入 personal-wechat-service，双击 一键启动.bat

自动安装依赖并启动在 3000 端口（微信需已登录）

五）在n8n放一个 WeChat Send 节点测试

在节点详情里面，点击创建新凭证

进行配置API和个人微信服务地址。

服务地址怎么填（最容易填错的地方）

注：某些 Linux 宿主无 host.docker.internal，请改用同网段容器或穿透地址。

微信服务类型选择个人微信自动化，

消息类型：图片；目标：文件传输助手等好友或群聊名称，中间用英文逗号隔开；内容随意

执行成功 → 微信收到 → 闭环打通

九、常见问答（精简版）

Q：个人微信和企业微信怎么选？  A：私域小规模触达/资料发送 → 个人微信；团队播报/日报/告警 → 企业微信。

Q：Docker 为什么不能用 localhost？  A：容器与宿主隔离。请用 host.docker.internal（Win）或穿透地址。

Q：URL 还是二进制更稳？  A：URL 更易复用与排查；本地/临时文件再用二进制。

Q：必须开随机延迟吗？  A：强烈建议，能显著降低“像机器人”的风险。

Q：为什么需要 API 密钥？  A：用于鉴权、使用统计与问题定位，以便持续优化服务体验。
