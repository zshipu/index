微软公布10月Visual Studio AI发展路线图，多项功能待改进调用代码知名企业visualstudio_网易订阅 - 知识铺

2025年10月5日
| 阅读

IT之家 10 月 5 日消息，微软公布了最新的 10 月份 Visual Studio 人工智能发展路线图，重点介绍了未来将在集成开发环境中推出的 AI 功能改进与新特性。

微软表示，将持续通过更智能、更快速、更直观的工具优化开发体验，但同时也强调这些内容代表的是正在进行的工作，并不保证在当月完成。

本月路线图涵盖多个方向，IT之家汇总如下：

新 Agents 智能体

微软开始在 Visual Studio 中引入远程 Agents，例如 GitHub Copilot 编码智能体，并计划进一步扩展此类原生体验。开发者可直接从聊天中启动 GitHub Copilot 会话。

新 Agents 与调试支持

微软正在优化模式切换方式，以支持更复杂的工作流。目前正在开发的新代理包括调试器代理。

Agents 模式与聊天改进

针对用户反馈，微软对代理模式和聊天功能进行大幅更新，包括：

  • 工具调用改进（支持动态调用）
  • 总结线程历史以便在对话中延续
  • 响应过程中切换代理模式
  • 在 Copilot 聊天中支持任务规划与待办管理

性能优化

性能和响应质量被列为最高优先级。改进措施主要包括：

  • 提升 Copilot 聊天的启动性能
  • Token 优化（减少使用量、提示缓存）
  • 调整代码搜索以确保调用效率

模型上下文协议（MCP）

微软希望开发者能够在 Visual Studio 中完整引入开发栈，并保持与现有安全与治理机制一致。本阶段工作包括：

  • MCP 完整规范支持
  • 改进用户体验
  • 加强治理控制（如组策略）
  • Windows MCP 注册表支持

更多模型接入

Visual Studio 将持续评估并引入最新模型，以提升开发体验。计划包括：

  • 聊天中的自动模型选择
  • 引入 Claude Sonnet 4.5
  • 引入 GPT-5 Codex

微软表示，其目标是让 Copilot 深度融入开发流程，不仅用于代码编写，还可支持搜索、Bug 修复、编写单元测试，以及提交与推送代码。
