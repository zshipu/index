IDE 领域最近出现了一些激烈的争论，尤其是围绕 Windsurf 和 Cursor IDE。一位与 Cursor IDE 相关的 YouTuber 发布了一段视频，该视频完全是试图阻止人们尝试 Windsurf 的偏见。在视频（此处）中，他声称要“测试”Windsurf，但花了更多时间抱怨它不起作用，似乎假装他无法弄清楚基本的编辑器。
看着他试图让观众反对 Windsurf，而不是提供公平和诚实的比较，我心碎了，我失去了曾经对他的尊重。

但让我们澄清一下： Windsurf是 Cursor 的强大替代品，而且我相信它是更好的选择，原因有几个。

为什么风帆冲浪超过光标？

Windsurf 和 Cursor 都构建在 VS Code 的免费开源基础上。
这就是为什么 Cursor 的 20 美元/月订阅费让人感觉昂贵——它本质上是内置 AI 的 VS Code 克隆，虽然这个价格可能对某些用户或特殊用例有意义，但并不是每个人都可以使用或合理。
并非每个开发人员都会持续使用人工智能，也并非许多编码任务总是需要人工智能。另一方面，Windsurf 提供了一种更实惠的方法，具有强大的免费套餐和每月 10 美元（或每年 8.33 美元/月）的付费计划。

以下是 Windsurf 免费套餐提供的内容：

  • 无限快速人工智能自动完成建议
  • 无限编辑器内 AI 聊天
  • 无限AI指令指令
  •  级联（只读模式）
  • 不对非许可数据进行培训

对于需要额外功能的开发人员，Windsurf 的付费计划提供更多功能：

  • 无限制地访问更大的 Codeium 模型
  • 无限制访问 GPT-4o、Claude Sonnet（聊天、Codeium 扩展）
  • 无限制访问 Supercomplete

 我的风帆之旅

正如我在上一篇文章“为什么 Windsurf 是您从未听说过的最好的免费 AI 代码编辑器”中讨论的那样，Windsurf 的_Cascade_流系统提供了无与伦比的体验，以一种方式集成了工具、知识和实时代码上下文： Cursor 的_Composer_功能无法匹配。 Windsurf 不仅提供更快、更准确的基于上下文的代码建议，还让我能够解决来自终端的错误，这极大地加快了我的工作流程。

此外，Windsurf 具有独特的能力，可以索引整个代码库并将其用作自动补全和代码建议的基础，这意味着建议始终根据存储库本身量身定制。
这种深度的集成是 Cursor 所缺乏的，因为它通常依赖于其在线知识库，有时会忽略特定于存储库的上下文。
事实上，这位 YouTuber 在他的视频中评论道：“Windsurf 适合不会编码的开发人员，而 Cursor 适合会编码的开发人员。”这与事实相差甚远。
Windsurf 的系统实际上假设开发人员的代码是正确的，并根据当前存储库的内容提出建议，而 Cursor 的 AI 经常绕过项目的结构。
对于寻求真正的定制和特定于上下文的支持的开发人员来说，在我看来，Windsurf 是为严肃的编码量身定制的。

定价争论及其重要性

该视频中的另一点是 YouTube 用户对定价的反应。当聊天中的观众询问 Windsurf 与 Cursor 相比的承受能力时，他大发雷霆，表示他对负担不起 Cursor 每月 20 美元订阅费用的开发者“毫不怜悯”。
他驳回了他们，建议如果他们付不起钱，就应该坚持使用更便宜的 IDE，并“找到一份合适的工作”。

他错过的是，并不是每个开发人员（或有抱负的开发人员）都能够在已经不断增长的费用清单上再增加一笔高昂的订阅费。
虽然 Cursor 和 Windsurf 都建立在 VS Code 的基础上，并具有更多内置的 AI，但并不是每个人都愿意为 Windsurf 以低得多的成本提供的功能支付溢价，甚至在某些情况下是免费的。
像 Windsurf 这样经济实惠且功能强大的工具可以为每个人打开大门，而不仅仅是拥有大量预算的开发人员，并且它们可以使所有类型的用户受益，从业余爱好者到专业人士。

给风帆冲浪一个公平的机会
