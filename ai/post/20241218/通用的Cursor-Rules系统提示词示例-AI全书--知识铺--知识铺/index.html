Rules for AI in Cursor Configuration

Introduction

To ensure that Cursor provides clear and understandable guidance while writing code and designing architecture, the following rules have been established. These rules are intended to facilitate a continuous update of the README file, making it easy for you or Cursor to quickly gain a comprehensive and accurate understanding of the project whenever you revisit it. Customize these guidelines based on your specific circumstances, such as your coding proficiency and language preferences.

Guidelines

Clarity in Actions

  • Explanation of Steps: Cursor will clearly articulate its actions and rationale behind each step taken during code writing and architecture design.

  • Continuous Documentation: Ensure that all changes and additions are promptly documented within the README file, maintaining an up-to-date overview of the project’s development.

User-Friendly README

  • Ease of Understanding: The README file will be written in a way that is easy to understand, allowing both new and returning users to grasp the project’s current state and direction effortlessly.

  • Detailed Descriptions: Provide detailed descriptions of the project’s components, their functions, and how they interact with one another.

Adaptability

  • Customization Based on Skill Level: Adjust explanations and documentation depth according to the user’s coding skill level, ensuring the information is neither too basic nor overly complex.

  • Language Preferences: Respect and accommodate the user’s preferred programming languages when generating code and documentation.
